Tổng quan về Reflection API

Khái niệm và mục đích sử dụng

Reflection là cơ chế mạnh mẽ trong Java cho phép chương trình tự động kiểm tra, phân tích và sửa đổi cấu trúc (metadata), hành vi của Class, Method, Field, Constructor trực tiếp tại thời điểm chạy (Runtime) mà không cần biết trước các định nghĩa này lúc biên dịch (Compile-time).

Reflection là xương sống để xây dựng nên hầu hết các thư viện và framework Java phổ biến như Spring (IoC Container), Hibernate (ORM mapping), Jackson (JSON parser), JUnit (Test runner).

Cơ chế hoạt động của Class Loader và JVM

Khi một class được nạp vào bộ nhớ RAM, JVM tự động tạo ra một đối tượng đại diện duy nhất kiểu java.lang.Class để chứa toàn bộ siêu dữ liệu (Metadata) của class đó:

[ Source Code.java ] ──> Compile ──> [ Bytecode .class ] ──> ClassLoader │ ▼ [ Reflection API ] ◄─────── [ JVM tạo duy nhất 1 Object java.lang.Class ]

23.1 Class Object (Đối tượng siêu dữ liệu)

Cách lấy đối tượng Class

Có 3 cách chính để lấy về đối tượng Class<?> đại diện cho một kiểu dữ liệu:

  1. Sử dụng trực tiếp cú pháp Class Literal: Class<User> clazz = User.class; (An toàn nhất thời điểm biên dịch).
  2. Lấy từ thực thể (Instance) qua getClass(): Class<?> clazz = user.getClass();
  3. Dùng chuỗi ký tự tên Class đầy đủ (Fully Qualified Name): Class<?> clazz = Class.forName("com.demo.User"); (Cách Spring Boot quét Component).
Ví dụ mã nguồn đọc Metadata
Class<?> clazz = Class.forName("com.demo.User");
System.out.println("Tên đầy đủ: " + clazz.getName());      // com.demo.User
System.out.println("Tên rút gọn: " + clazz.getSimpleName()); // User

23.2 Reflection Method (Kích hoạt phương thức động)

Khái niệm và cú pháp

Reflection cho phép bạn lấy ra các đối tượng đại diện cho hàm (java.lang.reflect.Method) và kích hoạt thực thi hàm đó động thông qua phương thức `invoke(instance, args)`.

Ví dụ mã nguồn thực thi hàm động
class User {
    public void sayHello(String message) {
        System.out.println("User says: " + message);
    }
}

public class Main {
    public static void main(String[] args) throws Exception {
        User userInstance = new User();
        Class<User> clazz = User.class;

        // Lấy method có tên sayHello và nhận tham số kiểu String
        Method method = clazz.getMethod("sayHello", String.class);

        // Kích hoạt hàm động trên userInstance
        method.invoke(userInstance, "Hello Reflection!"); // Kết quả: User says: Hello Reflection!
    }
}

23.3 Reflection Field (Phá vỡ đóng gói Encapsulation)

Truy cập và thay đổi Private Fields

Reflection có thể bypass (vượt qua) cơ chế bảo vệ truy cập private của Java. Bằng việc gọi `setAccessible(true)` trên đối tượng Field, ta có thể tự do đọc và ghi đè giá trị của biến private bên trong đối tượng mà không cần getter/setter.

Ví dụ mã nguồn đọc/ghi đè biến private
class SecretAgent {
    private String codeName = "007";
}

public class Main {
    public static void main(String[] args) throws Exception {
        SecretAgent agent = new SecretAgent();
        Class<?> clazz = agent.getClass();

        // Lấy field private tên codeName
        Field field = clazz.getDeclaredField("codeName");
        
        // Phá vỡ tính đóng gói
        field.setAccessible(true);

        // Đọc giá trị
        System.out.println(field.get(agent)); // Kết quả: 007

        // Ghi đè giá trị mới
        field.set(agent, "Double-O-Seven");
        System.out.println(field.get(agent)); // Kết quả: Double-O-Seven
    }
}

23.4 Reflection Constructor (Khởi tạo đối tượng động)

Cơ chế và ứng dụng của newInstance()

Thay vì khởi tạo đối tượng cứng bằng từ khóa new User(), Reflection cho phép lấy constructor động và gọi newInstance(args) để tạo đối tượng.

// Lấy constructor không tham số
Constructor<User> constructor = User.class.getConstructor();
User user = constructor.newInstance(); // Tạo thực thể động thành công!

// Lấy constructor có tham số String
Constructor<User> paramConstructor = User.class.getConstructor(String.class);
User userWithParam = paramConstructor.newInstance("Bang");

Ưu điểm & Nhược điểm của Reflection API

Ưu điểm nổi bật Nhược điểm & Rủi ro bảo mật
• Tính linh hoạt cực cao, cho phép viết các Framework cấu hình động.
• Hỗ trợ Dependency Injection tự động inject bean.
• Tự động ánh xạ dữ liệu nhị phân (JSON / Database Table). 		• Hiệu năng chậm hơn: JVM không thể tối ưu hóa và tối giản JIT compiler.
• Phá vỡ tính đóng gói (Encapsulation), tăng rủi ro bảo mật hệ thống.
• Khó kiểm soát lỗi tĩnh: Lỗi tên hàm/field sai chỉ phát hiện ở Runtime.

Các lỗi Exception thường gặp ở Runtime

  • `IllegalAccessException`: Quên gọi field.setAccessible(true) trước khi lấy/gán giá trị của một biến private.
  • `NoSuchMethodException` / `NoSuchFieldException`: Viết sai chính tả tên method/field hoặc sai kiểu tham số truyền vào khi dùng Reflection.
  • `InvocationTargetException`: Lỗi xảy ra khi method được kích hoạt thông qua `invoke()` tự bản thân nó ném ra một Exception bên trong.
