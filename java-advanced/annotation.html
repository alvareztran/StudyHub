Annotations & Metadata

Khái niệm siêu dữ liệu (Metadata)

Annotation là một hình thức siêu dữ liệu (metadata) được gắn trực tiếp vào các phần tử của chương trình như Class, Method, Field, Parameter. Nó cung cấp các thông tin khai báo bổ sung mà không làm thay đổi logic hoạt động trực tiếp của mã nguồn.

Dữ liệu nhãn dán Annotation sẽ được các công cụ biên dịch (Compiler) hoặc các bộ thư viện chạy runtime đọc ra thông qua cơ chế Reflection để thực thi các hành động tương ứng cấu hình.

Cơ chế hoạt động bên trong của Annotation
[ Source Code ] (Ví dụ: @Override, @Entity) │ ▼ (Biên dịch) [ Bytecode .class ] ──> Lưu trữ các thẻ mô tả Annotation │ ▼ (Lớp nạp ClassLoader & JVM / Framework chạy) [ Java Reflection API ] ──> Quét các chú thích ──> Thực thi logic tự động

22.1 Built-in Annotations (Annotations có sẵn của Java)

Khái niệm và danh sách các Built-in phổ biến

Built-in Annotations là các chú thích do JDK phát triển sẵn để kiểm soát lỗi tĩnh thời điểm compile.

Annotation Mục đích sử dụng Cơ chế kiểm soát của Compiler
@Override Đánh dấu phương thức ghi đè phương thức lớp cha. Báo lỗi biên dịch ngay lập tức nếu viết sai tên hàm hoặc sai chữ ký tham số.
@Deprecated Đánh dấu các API cũ, lỗi thời, không khuyến cáo dùng. Sinh cảnh báo gạch ngang tên hàm (warning) khi lập trình viên cố tình gọi hàm này.
@SuppressWarnings Tắt các cảnh báo không cần thiết của compiler. Ví dụ: @SuppressWarnings("unchecked") tắt cảnh báo ép kiểu generic không an toàn.
@FunctionalInterface Đảm bảo interface chỉ có duy nhất một hàm abstract. Ngăn chặn việc khai báo hàm abstract thứ hai, hỗ trợ liên kết biểu thức Lambda.

22.2 Custom Annotations (Tự định nghĩa Annotations)

Cách định nghĩa Annotation

Để tạo một Custom Annotation, ta sử dụng từ khóa **`@interface`** kết hợp với các **Meta-Annotations** để cấu hình tầm vực:

import java.lang.annotation.*;

@Retention(RetentionPolicy.RUNTIME) // Tồn tại khi chạy chương trình để quét qua Reflection
@Target(ElementType.METHOD)        // Chỉ cho phép gắn trên Method
public @interface Transactional {
    String value() default "dbTransaction";
    int timeout() default 30;
}
Khảo sát chi tiết Meta-Annotations
  • `@Target`: Xác định vị trí được phép gắn Annotation (ví dụ: ElementType.TYPE cho class/interface, ElementType.FIELD cho biến thuộc tính, ElementType.METHOD cho hàm).
  • `@Retention`: Xác định vòng đời tồn tại của Annotation:
    RetentionPolicy Vòng đời chi tiết
    SOURCEChỉ tồn tại ở file nguồn, compiler sẽ xóa bỏ hoàn toàn khi dịch sang tệp `.class` (ví dụ: Lombok `@Getter`, `@Setter`).
    CLASSTồn tại ở tệp `.class` nhưng bị JVM loại bỏ lúc nạp vào RAM chạy ứng dụng.
    RUNTIMETồn tại vĩnh viễn lúc chạy ứng dụng. Cho phép Reflection API đọc động dữ liệu (95% Framework Spring/Hibernate dùng chính sách này).
Cơ chế quét cấu trúc động bằng Reflection

Ví dụ minh họa cách thức Spring/Hibernate quét qua Reflection để đọc Annotation và thực thi mã:

// 1. Gắn nhãn trên Class UserService
@Author(name = "Bang")
public class UserService { }

// 2. Viết Engine quét lúc chạy chương trình
public class AnnotationScanner {
    public static void main(String[] args) {
        Class<UserService> clazz = UserService.class;
        
        // Kiểm tra xem class có được gắn nhãn @Author không
        if (clazz.isAnnotationPresent(Author.class)) {
            Author author = clazz.getAnnotation(Author.class);
            System.out.println("Tác giả viết class này là: " + author.name());
            // Kết quả: Tác giả viết class này là: Bang
        }
    }
}

Các lỗi cấu hình nghiêm trọng của Junior khi viết Annotation

  • Quên đặt RetentionPolicy.RUNTIME: Khai báo custom annotation nhưng thiếu `@Retention` hoặc đặt thành `RetentionPolicy.CLASS`. Khi chạy, Reflection trả về kết quả rỗng (null) khiến framework không hoạt động.
  • Sai ElementType Target: Gắn nhãn lên thuộc tính (Field) nhưng ở định nghĩa chỉ khai báo `@Target(ElementType.METHOD)`. Compiler sẽ chặn lỗi biên dịch.
  • Thiếu giá trị bắt buộc: Các thuộc tính trong `@interface` không có từ khóa `default` thì bắt buộc phải gán giá trị khi sử dụng, nếu không compiler sẽ báo lỗi.
