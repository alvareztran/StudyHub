18.1 Cú pháp Lambda (Lambda Syntax)

Khái niệm và mục đích sử dụng

Lambda Expression (Biểu thức Lambda) giới thiệu từ **Java 8** là cú pháp rút gọn đặc biệt dùng để đại diện cho một thể hiện (Instance) của một **Anonymous Class (Lớp vô danh)** mà lớp đó chỉ chứa duy nhất một phương thức trừu tượng.

Sự cải tiến so với cách viết Anonymous Class truyền thống:

// Trước Java 8 (Cách viết Anonymous Class rườm rà):
Runnable runnable = new Runnable() {
    @Override
    public void run() {
        System.out.println("Hello World");
    }
};

// Từ Java 8 (Cú pháp Lambda siêu gọn):
Runnable runnable = () -> System.out.println("Hello World");
Cơ chế hoạt động bên trong (invokedynamic)

Trình biên dịch Java (Compiler) không xử lý Lambda bằng cách dịch nó thành class vô danh độc lập để tránh gánh nặng sinh tệp tin cấu trúc .class làm cồng kềnh Metaspace.

Thay vào đó, nó tạo ra một chỉ thị Bytecode là `invokedynamic`. Khi JVM thực thi Bytecode này ở Runtime, nó dùng cơ chế LambdaMetafactory để tạo động đối tượng thực thi trực tiếp, giúp tiết kiệm bộ nhớ Heap tối đa và đẩy cao hiệu năng xử lý.

Cú pháp và quy tắc
(parameters) -> { statements; }

Quy tắc giản lược cú pháp:

  • Không tham số: Bắt buộc dùng ngoặc tròn trống: () -> System.out.println("No params");
  • 1 tham số: Có thể lược bỏ cặp ngoặc tròn: name -> System.out.println(name);
  • Nhiều tham số: Bắt buộc dùng cặp ngoặc: (a, b) -> a + b;
  • Nhiều dòng lệnh: Bắt buộc bao quanh bằng cặp ngoặc nhọn {} và có từ khóa return nếu phương thức có kiểu trả về.

18.2 Giao diện đơn hàm (Functional Interface)

Khái niệm và quy tắc

Functional Interface là một interface chỉ sở hữu **duy nhất một phương thức trừu tượng (abstract method)**. Biểu thức Lambda chỉ có thể liên kết và hoạt động dựa trên cấu trúc Functional Interface.

Ta sử dụng annotation **`@FunctionalInterface`** trước interface để ép buộc trình biên dịch báo lỗi compile ngay lập tức nếu vô tình khai báo thêm phương thức trừu tượng thứ hai.

@FunctionalInterface
public interface Calculator {
    int calculate(int a, int b); // Chỉ 1 abstract method duy nhất
}
Các Functional Interface phổ biến cung cấp sẵn của Java 8

Java cung cấp sẵn gói thư viện java.util.function chứa các Functional Interface kinh điển phục vụ xử lý dữ liệu động:

Interface Đầu vào (Input) Đầu ra (Output) Ví dụ ứng dụng thực tế
Predicate<T>TbooleanKiểm tra điều kiện: number -> number % 2 == 0
Function<T, R>TRChuyển đổi dữ liệu: str -> str.length()
Consumer<T>TvoidTiêu thụ dữ liệu, in ra: item -> log.info(item)
Supplier<T>NoneTCung cấp giá trị mới: () -> Math.random()

18.3 Method Reference (Tham chiếu phương thức)

Khái niệm và cú pháp

Method Reference sử dụng ký hiệu toán tử hai dấu hai chấm **`::`**, là cú pháp viết ngắn gọn của biểu thức Lambda khi Lambda đó chỉ làm đúng một nhiệm vụ duy nhất là gọi lại một phương thức sẵn có.

Ví dụ so sánh:

// Dùng Lambda:
names.forEach(name -> System.out.println(name));

// Dùng Method Reference (Rút gọn tối đa):
names.forEach(System.out::println);
Phân loại các hình thức Method Reference
Loại tham chiếu Cú pháp Ví dụ cụ thể
Static MethodClassName::staticMethodInteger::parseInt (tương đương s -> Integer.parseInt(s))
Instance Methodobject::instanceMethodSystem.out::println (tương đương x -> System.out.println(x))
Class Instance MethodClassName::instanceMethodString::toUpperCase (tương đương s -> s.toUpperCase())
Constructor ReferenceClassName::newUser::new (tương đương () -> new User())
