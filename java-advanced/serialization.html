Tổng quan về Serialization

Khái niệm và mục đích sử dụng

Serialization (Tuần tự hóa) là cơ chế trong Java cho phép chuyển đổi trạng thái của một đối tượng (Object) thành một chuỗi byte nhị phân (Byte Stream). Chuỗi byte này có thể lưu trữ xuống ổ đĩa, lưu vào bộ đệm Cache hoặc truyền gửi qua mạng.

Deserialization (Giải tuần tự hóa) là quá trình ngược lại: JVM tiếp nhận chuỗi byte để tái tạo ngược trở lại trạng thái đối tượng gốc trong bộ nhớ Heap.

Cơ chế hoạt động bên trong

Quy trình vận chuyển trạng thái ô nhớ qua dòng dữ liệu nhị phân:

[ Java Object (Heap) ] ──> ObjectOutputStream ──> Byte Stream ──> [ Lưu trữ file/mạng ] │ (Dịch chuyển) [ Java Object (Heap) ] ◄── ObjectInputStream ◄── Byte Stream ◄── [ Đọc nhị phân ]
Ví dụ ứng dụng trong dự án thực tế
  • Lưu trữ HTTP Session: Khi thực hiện cấu hình Session phân tán bằng Redis (Spring Session + Redis), dữ liệu người dùng đăng nhập phải được serialize trước khi lưu trữ vào DB Redis.
  • Truyền nhận tin nhắn (Message Broker): Gửi nhận DTO đối tượng qua Kafka, RabbitMQ, ActiveMQ.

21.1 Interface Serializable

Khái niệm Marker Interface

Interface java.io.Serializable là một **Marker Interface** (giao diện đánh dấu) đặc biệt - nó hoàn toàn **không chứa bất kỳ phương thức trừu tượng nào**. Nó chỉ đóng vai trò cấp quyền, thông báo cho JVM biết rằng Class này được phép tuần tự hóa.

Nếu bạn cố tình gọi ghi file nhị phân một đối tượng không implement Serializable, JVM lập tức ném lỗi sập luồng: **`java.io.NotSerializableException`**.

Mã số định danh phiên bản: serialVersionUID

Trong mỗi class implement Serializable, ta cần khai báo hằng số:

private static final long serialVersionUID = 1L;

JVM sử dụng mã số này để xác thực tính đồng nhất phiên bản giữa lớp gửi (lúc serialize) và lớp nhận (lúc deserialize). Nếu mã này khác nhau (do thay đổi thuộc tính class mà không quản lý phiên bản), JVM ném lỗi InvalidClassException.

Ví dụ mã nguồn ghi/đọc File nhị phân
import java.io.*;

class User implements Serializable {
    private static final long serialVersionUID = 1L;
    private String name;
    public User(String name) { this.name = name; }
    @Override
    public String toString() { return "User: " + name; }
}

public class Main {
    public static void main(String[] args) throws Exception {
        // 1. Serialization (Ghi đối tượng xuống file)
        try (ObjectOutputStream out = new ObjectOutputStream(new FileOutputStream("user.dat"))) {
            out.writeObject(new User("Bang"));
        }

        // 2. Deserialization (Đọc đối tượng từ file)
        try (ObjectInputStream in = new ObjectInputStream(new FileInputStream("user.dat"))) {
            User user = (User) in.readObject();
            System.out.println(user); // Kết quả: User: Bang
        }
    }
}

21.2 Từ khóa transient (Loại trừ thuộc tính nhạy cảm)

Khái niệm và mục đích sử dụng

Từ khóa `transient` dùng để đánh dấu một biến thuộc tính sẽ **hoàn toàn bị bỏ qua (không ghi dữ liệu)** trong quá trình tuần tự hóa.

Ứng dụng thực tế: Ngăn chặn rò rỉ dữ liệu nhạy cảm hoặc bỏ qua các đối tượng không thể tuần tự hóa vật lý (như Database Connection, Socket, Thread, I/O Streams):

public class UserSession implements Serializable {
    private static final long serialVersionUID = 1L;
    
    private String username; // Được serialize
    private transient String password; // Bị BỎ QUA không serialize
    private transient Connection dbConnection; // Bị BỎ QUA tránh lỗi crash app
}
Giá trị nhận lại sau khi giải tuần tự hóa (Deserialization)

Khi giải tuần tự hóa ngược lại, các thuộc tính đánh dấu transient sẽ tự động nhận giá trị mặc định của kiểu dữ liệu tương ứng trong bộ nhớ:

  • Kiểu đối tượng (String, Object): Nhận giá trị mặc định là **`null`**.
  • Kiểu số nguyên thủy (int, long): Nhận giá trị mặc định là **`0`**.
  • Kiểu logic nguyên thủy (boolean): Nhận giá trị mặc định là **`false`**.

Các lỗi bảo mật & Nghiệp vụ thường gặp khi đi làm

  • Không serialize các thuộc tính tham chiếu con: Nếu Class cha implement Serializable chứa một đối tượng Class con bên trong, thì bắt buộc Class con đó cũng phải implement Serializable. Nếu không, quá trình ghi file sẽ ném ra lỗi NotSerializableException.
  • Rò rỉ thông tin mật khẩu bảo mật: Quên không khai báo transient cho các thuộc tính nhạy cảm như password, token, pin code dẫn đến việc các thông tin này ghi thô ở dạng nhị phân xuống ổ đĩa, dễ bị hacker khai thác.
  • deploy phiên bản lỗi: Sửa đổi cấu trúc Class (thêm bớt field) mà không cấu hình cố định serialVersionUID, làm cho dữ liệu cũ đã lưu ở DB Cache/Redis không thể đọc được nữa khi server khởi động phiên bản code mới.
