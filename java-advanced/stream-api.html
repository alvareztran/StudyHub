Stream API & Stream Pipeline

Khái niệm và mục đích sử dụng

Stream API (giới thiệu trong **Java 8**) là công cụ cho phép xử lý các tập dữ liệu Collection theo phong cách **khai báo (Declarative Programming)** thay thế hoàn toàn cho cách viết duyệt vòng lặp truyền thống **mệnh lệnh (Imperative Programming)**.

Lợi ích: Giúp mã nguồn cực kỳ ngắn gọn, trực quan, loại bỏ các biến cờ cồng kềnh và giảm thiểu tối đa các lỗi logic chỉ mục.

Cơ chế hoạt động của đường ống dẫn dữ liệu (Stream Pipeline)

Một chu trình xử lý Stream hoàn chỉnh bao gồm 3 bước bắt buộc được thực hiện theo cơ chế pipeline ống nước:

[ Source Collection ] (Nguồn cấp dữ liệu thô) │ ▼ (1. Khởi tạo Stream: .stream() ) [ Intermediate Operation ] (filter, map, sorted, distinct...) -> Trả về Stream mới │ ▼ (2. Hỗ trợ Lazy Evaluation - Không chạy cho đến khi có Terminal Operation) [ Terminal Operation ] (collect, forEach, reduce, count...) -> Kết thúc & sinh kết quả

Tính chất cốt lõi:

  • Không lưu trữ dữ liệu: Stream chỉ là một ống dẫn, không sở hữu ô nhớ lưu trữ phần tử.
  • Không biến đổi Collection gốc: Kết quả luôn được đẩy ra một Collection hoặc giá trị mới.
  • Lazy Evaluation (Đánh giá lười biếng): Các hàm trung gian (Intermediate) không hề xử lý cho tới khi hàm cuối (Terminal) được kích hoạt.

19.1 filter() & 19.2 map()

1. filter() - Phép lọc điều kiện

Hàm filter() nhận vào tham số là một Predicate<T> (kiểm tra boolean) để sàng lọc và giữ lại các phần tử thỏa mãn.

// Ví dụ: Lọc danh sách User đang kích hoạt (Active)
List<User> activeUsers = users.stream()
                              .filter(User::isActive) // Thỏa mãn điều kiện thì giữ lại
                              .toList();
2. map() - Phép ánh xạ chuyển đổi dữ liệu

Hàm map() nhận vào một Function<T, R> để biến đổi kiểu dữ liệu của từng phần tử từ cấu trúc này sang cấu trúc khác.

// Ví dụ: Ánh xạ chuyển đổi từ danh sách Entity sang DTO để trả về Client
List<UserDTO> dtos = users.stream()
                          .map(user -> new UserDTO(user.getId(), user.getName()))
                          .toList();

19.3 reduce() (Phép thu gọn/gộp)

Khái niệm và cơ chế vận hành

Phương thức reduce() thực hiện gộp toàn bộ các phần tử trong Stream lại với nhau thông qua một phép toán tích lũy để tạo ra một giá trị duy nhất.

Cú pháp:

stream.reduce(identity, accumulator)
  • identity: Giá trị khởi tạo mặc định ban đầu.
  • accumulator: Hàm tích lũy thực hiện cộng dồn giá trị tích lũy với phần tử tiếp theo.

Cơ chế tích lũy tính tổng doanh thu hóa đơn:

Doanh thu đơn hàng: [ 10.0, 20.0, 50.0 ] với identity = 0.0 Bước 1: 0.0 + 10.0 = 10.0 Bước 2: 10.0 + 20.0 = 30.0 Bước 3: 30.0 + 50.0 = 80.0 (Kết quả cuối cùng thu gọn)
double totalRevenue = orders.stream()
                            .map(Order::getTotal)
                            .reduce(0.0, Double::sum);

19.4 collect() & 19.5 groupingBy()

1. collect() - Thu gom kết quả cuối đường ống

collect() là Terminal Operation phổ biến nhất dùng để đóng gói toàn bộ luồng Stream sau xử lý vào các cấu trúc dữ liệu mong muốn thông qua các Collector tiện ích:

Set<String> emails = users.stream()
                          .map(User::getEmail)
                          .collect(Collectors.toSet()); // Gom vào Set để loại bỏ trùng lặp
2. groupingBy() - Gom nhóm phân loại dữ liệu (SQL GROUP BY)

Bộ thu gom Collectors.groupingBy() phân nhóm các phần tử trong Stream dựa trên thuộc tính chung và trả về kết quả dạng Map<K, List<T>>.

Ví dụ: Nhóm danh sách đơn hàng theo trạng thái thanh toán:

Map<OrderStatus, List<Order>> ordersByStatus = orders.stream()
    .collect(Collectors.groupingBy(Order::getStatus));
// Kết quả trả về:
// Map { PENDING -> [Order1, Order2], COMPLETED -> [Order3] }

Lời khuyên tối ưu: Đối với các tác vụ xử lý tính toán số học siêu hiệu năng yêu cầu latency cực thấp nên chọn For Loop. Với các tác vụ thao tác dữ liệu, ánh xạ nghiệp vụ kinh doanh, luôn ưu tiên **Stream**.

Các lỗi nghiêm trọng thường gặp khi làm Stream

1. Lỗi tái sử dụng luồng Stream đã đóng

Một Stream trong Java chỉ được sử dụng duy nhất một lần. Khi một Terminal Operation đã chạy qua, Stream đó lập tức bị đóng lại vĩnh viễn.

Stream<String> stream = names.stream();
long count = stream.count(); // Stream bị đóng tại đây
stream.forEach(System.out::println); // Ném lỗi sập app: IllegalStateException
2. Lỗi quên hàm kết thúc (Terminal Operation)

Do cơ chế **Lazy Evaluation**, các hàm trung gian chỉ là khai báo đường ống, chúng sẽ hoàn toàn không thực thi nếu không có hàm kết thúc ở cuối.

numbers.stream().filter(n -> n > 10); // Không có bất kỳ phần tử nào được lọc hay in ra!
