15.1 equals()

Khái niệm và mục đích sử dụng

Phương thức equals() dùng để so sánh **nội dung giá trị thực tế** (Value/State) của hai đối tượng có tương đồng hay không.

Sự khác biệt cốt lõi giữa so sánh bằng toán tử == và phương thức equals():

  • Toán tử ==: So sánh địa chỉ con trỏ vùng nhớ (Reference) của hai đối tượng trên Heap. Chỉ trả về true khi cả hai biến trỏ chung về cùng 1 ô nhớ.
  • Phương thức equals(): Đi sâu so sánh thuộc tính, trạng thái nội dung của đối tượng (sau khi đã được ghi đè logic).
Cơ chế hoạt động bên trong

Mặc định, nếu bạn không override phương thức này, lớp java.lang.Object triển khai code thô như sau:

public boolean equals(Object obj) {
    return (this == obj); // Mặc định vẫn chỉ so sánh địa chỉ bộ nhớ
}

Do đó, để so sánh giá trị các thuộc tính của lớp tự định nghĩa, ta **bắt buộc phải ghi đè (Override) phương thức equals()**.

Cú pháp và quy tắc

Tạo 2 đối tượng độc lập có thuộc tính trùng khớp hoàn toàn:

User u1 = new User("Bang");
User u2 = new User("Bang");
System.out.println(u1.equals(u2)); // Trả về false nếu chưa override, và true nếu đã override chuẩn
Ví dụ mã nguồn đơn giản

Triển khai ghi đè equals() an toàn và chuẩn mực:

import java.util.Objects;

class User {
    private String name;
    public User(String name) { this.name = name; }

    @Override
    public boolean equals(Object obj) {
        if (this == obj) return true; // Cùng địa chỉ ô nhớ thì chắc chắn bằng nhau
        if (obj == null || getClass() != obj.getClass()) return false; // Khác class hoặc rỗng

        User user = (User) obj; // Ép kiểu
        return Objects.equals(name, user.name); // So sánh thuộc tính name an toàn null
    }
}
Ví dụ thực tế trong dự án

Trong các lớp thực thể (Entity) quản lý bởi ORM Hibernate / JPA, phương thức equals() thường được cấu hình so sánh duy nhất thuộc tính khóa chính **`id`**:

@Override
public boolean equals(Object obj) {
    if (this == obj) return true;
    if (!(obj instanceof User)) return false;
    User other = (User) obj;
    return id != null && id.equals(other.getId());
}

15.2 hashCode()

Khái niệm và mục đích sử dụng

Phương thức hashCode() trả về một giá trị số nguyên 32-bit đại diện duy nhất cho trạng thái của đối tượng đó tại thời điểm chạy.

Mục đích sử dụng:

  • Giúp tăng tốc độ tìm kiếm phần tử cực nhanh trong các cấu trúc dữ liệu bảng băm (Hashing) như HashSet, HashMap, HashTable.
Cơ chế hoạt động bên trong cấu trúc Hashing

Khi thêm một cặp khóa vào map: map.put(user, "Admin");, JVM thực thi:

user (Key) ──> Gọi user.hashCode() ──> Tính bucket index ↓ Truy cập ô bucket ↓ Nếu có node: Gọi equals() đối chiếu
Quy tắc liên kết tối thượng (Contract equals & hashCode)

⚠️ Quy tắc phỏng vấn vàng:
- Nếu a.equals(b) == true thì bắt buộc giá trị hash a.hashCode() == b.hashCode().
- Nếu a.hashCode() == b.hashCode() thì không bắt buộc a.equals(b) phải trả về true (Hiện tượng đụng độ mã băm).

Ví dụ mã nguồn đơn giản
import java.util.Objects;

class User {
    private String name;
    public User(String name) { this.name = name; }

    @Override
    public int hashCode() {
        return Objects.hash(name); // Tính mã hash dựa trên giá trị của name
    }
}

15.3 toString()

Khái niệm và mục đích sử dụng

Phương thức toString() trả về chuỗi văn bản đại diện mô tả ngắn gọn thông tin trạng thái thuộc tính của đối tượng.

Mục đích: Hỗ trợ ghi log thông tin nghiệp vụ và debug theo dõi lỗi lập trình trực quan.

Cơ chế hoạt động bên trong

Mặc định nếu không ghi đè, hàm toString() của Object trả về chuỗi định dạng thô khan hiếm thông tin:

public String toString() {
    return getClass().getName() + "@" + Integer.toHexString(hashCode());
}
// Ví dụ: User@5e91993f
Ví dụ mã nguồn đơn giản
class User {
    private String name;
    private int age;

    public User(String name, int age) {
        this.name = name;
        this.age = age;
    }

    @Override
    public String toString() {
        return "User{name='" + name + "', age=" + age + "}";
    }
}
Hỗ trợ của thư viện Lombok trong dự án thực tế

Trong môi trường đi làm doanh nghiệp, lập trình viên không viết code boiler-plate này thủ công, mà sử dụng Annotation `@Data` hoặc `@ToString` của thư viện Lombok để tự động sinh mã nguồn toString, equals, hashCode lúc build:

import lombok.Data;

@Data // Tự động sinh getter, setter, toString, equals, hashCode
public class User {
    private String name;
    private int age;
}

15.4 clone()

Khái niệm và mục đích sử dụng

Phương thức clone() dùng để nhân bản sao chép, tạo ra một thực thể đối tượng mới có dữ liệu y hệt đối tượng hiện tại nhưng nằm ở địa chỉ bộ nhớ Heap độc lập.

Cú pháp và quy tắc sử dụng

Để gọi được hàm clone, một Class bắt buộc phải:

  1. Khai báo thực thi Interface đánh dấu (marker interface): Cloneable. Nếu không có, JVM ném lỗi CloneNotSupportedException lúc chạy.
  2. Ghi đè phương thức clone() đổi phạm vi truy cập từ protected thành public.
class User implements Cloneable {
    private String name;
    public User(String name) { this.name = name; }

    @Override
    public Object clone() throws CloneNotSupportedException {
        return super.clone();
    }
}

Shallow Copy (Sao chép nông) và Deep Copy (Sao chép sâu)

Đây là kiến trúc phân tích cực kỳ quan trọng khi nhân bản đối tượng phức tạp chứa các tham chiếu đối tượng con bên trong:

1. Shallow Copy (Sao chép nông)

Chỉ nhân bản đối tượng lớp ngoài. Các đối tượng tham chiếu bên trong vẫn dùng chung địa chỉ con trỏ với đối tượng gốc.

user1 (0x10) ──> AddressObj (0x99) user2 (0x20) ──> AddressObj (0x99) (Dùng chung địa chỉ đối tượng con Address)

Hậu quả: Nếu thay đổi thông tin địa chỉ của user2 thì địa chỉ của user1 cũng bị sửa đổi theo.

2. Deep Copy (Sao chép sâu)

Nhân bản toàn diện từ lớp ngoài vào tận các đối tượng tham chiếu nằm sâu bên trong. Tạo ra các thực thể đối tượng độc lập hoàn toàn.

user1 (0x10) ──> AddressObj1 (0x99) user2 (0x20) ──> AddressObj2 (0x77) (Hai địa chỉ đối tượng con Address khác biệt hoàn toàn)

Cảnh báo thực tế: Do cơ chế `clone()` mặc định của Java chỉ thực hiện sao chép nông (Shallow Copy) và dễ gây lỗi rò rỉ, các dự án phần mềm hiện đại **không khuyến khích sử dụng clone()**. Thay vào đó, lập trình viên ưu tiên sử dụng các phương pháp thay thế an toàn hơn:

  • Sử dụng **Copy Constructor**: public User(User other) { ... }
  • Sử dụng mẫu thiết kế **Builder Pattern**
  • Sử dụng các thư viện mapping đối tượng tự động như **MapStruct** hoặc **ModelMapper**.
