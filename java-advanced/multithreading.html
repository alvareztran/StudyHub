Tổng quan về Multithreading

Khái niệm và mục đích sử dụng

Multithreading (Đa luồng) là kỹ thuật cho phép một ứng dụng xử lý song song (concurrency) nhiều tác vụ cùng một lúc bằng cách phân chia công việc cho nhiều phân luồng (Thread) chạy độc lập dưới sự điều phối của CPU.

Ứng dụng thực tế: Xử lý nhiều request HTTP đồng thời từ Client trên Web Server (Tomcat, Jetty), download tập tin lớn ở chế độ background, gửi email hàng loạt không gây nghẽn giao diện người dùng.

Mô hình bộ nhớ Thread trong JVM

Khi một luồng mới được tạo ra, JVM cấp phát riêng cho nó một phân vùng nhớ độc lập gọi là **Thread Stack** (để lưu biến cục bộ, tham số và khung gọi hàm LIFO). Tuy nhiên, toàn bộ các Thread này đều **chia sẻ chung phân vùng Heap Memory** và **Method Area** của Process cha:

[ JVM Process ] ──────── Shared Heap & Method Area ──────┐ │ │ ├── [ Thread 1 ] ──> Cấp phát riêng: Thread Stack 1 │ ├── [ Thread 2 ] ──> Cấp phát riêng: Thread Stack 2 │ └── [ Thread 3 ] ──> Cấp phát riêng: Thread Stack 3 ┘
Vòng đời của Thread (Thread Lifecycle)

Một Thread trong Java trải qua các trạng thái tuần tự được quản lý bởi Thread Scheduler của hệ điều hành:

  1. NEW: Khởi tạo đối tượng Thread nhưng chưa chạy start().
  2. RUNNABLE: Luồng sẵn sàng chạy và đợi hệ điều hành cấp CPU Core.
  3. BLOCKED / WAITING: Luồng bị tạm dừng do chờ khóa đồng bộ (monitor lock) hoặc đợi tín hiệu notify từ luồng khác.
  4. TERMINATED: Phương thức run() kết thúc hoàn toàn vòng đời.

24.1 Class Thread & 24.2 Interface Runnable

1. Kế thừa Class Thread

Tạo luồng bằng cách tạo class con kế thừa lớp java.lang.Thread và ghi đè phương thức run().

class MyThread extends Thread {
    @Override
    public void run() {
        System.out.println("Thread đang chạy");
    }
}
// Chạy luồng:
MyThread t = new MyThread();
t.start(); // Tạo OS Thread động và đẩy vào Thread Scheduler

⚠️ Sai lầm chí mạng: Gọi trực tiếp thread.run() thay vì thread.start(). Phương thức run() chỉ thực thi như một hàm thông thường trên luồng hiện tại (Main Thread), hoàn toàn không khởi tạo luồng song song mới nào.

2. Implement Interface Runnable (Khuyên dùng)

Do Java không hỗ trợ đa kế thừa, nếu Class của bạn đã kế thừa một class cha khác thì không thể kế thừa Thread. Giải pháp tối ưu là implement giao diện đơn hàm **`Runnable`**:

// Sử dụng cú pháp Lambda tối giản:
Thread t = new Thread(() -> {
    System.out.println("Tác vụ Runnable chạy song song");
});
t.start();

24.3 Callable & 24.4 Future (Thu thập kết quả trả về)

1. Interface Callable - Trả kết quả và ném Exception

Khác với Runnable chỉ thực thi tác vụ kiểu void và không thể ném checked exception, interface Callable<V> cho phép trả về một kết quả kiểu dữ liệu động và ném lỗi an toàn:

@FunctionalInterface
public interface Callable<V> {
    V call() throws Exception;
}
2. Interface Future - Phiếu thu hoạch kết quả tương lai

Khi gửi (submit) một Callable vào dịch vụ quản lý luồng ExecutorService, bạn nhận về một đối tượng **`Future`** đóng vai trò như một "phiếu chờ nhận món ăn":

import java.util.concurrent.*;

public class FutureDemo {
    public static void main(String[] args) throws Exception {
        ExecutorService executor = Executors.newSingleThreadExecutor();

        Callable<Integer> task = () -> {
            Thread.sleep(2000); // Giả lập tính toán nặng mất 2 giây
            return 42;
        };

        // Gửi tác vụ vào luồng background và nhận phiếu Future
        Future<Integer> future = executor.submit(task);
        System.out.println("Đang làm việc khác song song...");

        // Lấy kết quả (hàm get() sẽ chặn luồng hiện tại cho tới khi task hoàn thành)
        Integer result = future.get(); 
        System.out.println("Kết quả thu được: " + result); // 42
        
        executor.shutdown(); // Bắt buộc tắt executor để tránh rò rỉ tài nguyên
    }
}
Các phương thức điều khiển chính của Future
  • get(): Chặn luồng (blocking) để lấy kết quả.
  • isDone(): Kiểm tra tác vụ đã xử lý xong hay chưa (không gây block).
  • cancel(true): Gửi tín hiệu hủy bỏ thực thi tác vụ.

Bảng so sánh Thread vs Runnable vs Callable

Tiêu chí so sánh Kế thừa Thread Implement Runnable Implement Callable
Đa kế thừa ClassKhông (đã tốn lượt extends)Có (tự do extends class khác)Có (tự do extends class khác)
Kết quả trả vềKhông (trả về void)Không (trả về void)Có (trả về kiểu Generic V)
Ném ExceptionKhôngKhôngCó (throws Checked Exception)
Cách thức thực thistart() trực tiếpĐóng gói vào new Thread(runnable)Gửi vào ExecutorService.submit()

Các lỗi nghiệp vụ nguy hiểm khi lập trình đa luồng

  • Không tắt ExecutorService: Khởi tạo các thread pool (ví dụ: Executors.newFixedThreadPool()) nhưng quên gọi shutdown() khi dừng ứng dụng. Các luồng nền vẫn tiếp tục duy trì hoạt động làm rò rỉ bộ nhớ nghiêm trọng (Memory Leak) khiến hệ điều hành quá tải.
  • Tạo luồng mới vô tội vạ: Viết vòng lặp tạo mới thực thể Thread thủ công (new Thread().start()) hàng nghìn lần làm sập RAM do chi phí cấp phát bộ nhớ Stack cho mỗi Thread quá lớn. Giải pháp luôn là sử dụng **Thread Pool** thông qua `ExecutorService`.
