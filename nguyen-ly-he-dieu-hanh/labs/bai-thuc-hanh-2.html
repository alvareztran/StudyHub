BÀI THỰC HÀNH BUỔI 2 - CƠ BẢN VỀ LẬP TRÌNH SHELL

Mục tiêu: Làm quen với cách viết, cấp quyền và thực thi một Shell Script; định nghĩa và sử dụng biến; thực hiện các phép toán số học; sử dụng lệnh đọc dữ liệu và nhận đối số truyền vào từ dòng lệnh.

1. Tạo một Shell Script

Shell Script thực chất chỉ là một file văn bản thường chứa một tập hợp nhiều câu lệnh Linux được sắp xếp theo thứ tự thực thi từ trên xuống dưới.

Thay vì phải nhập thủ công từng câu lệnh:

echo Hello
date
pwd

Chúng ta lưu các dòng lệnh này vào một file văn bản và yêu cầu Shell biên dịch và thực thi nó.

Bước 1: Tạo file script

Có thể sử dụng bất kỳ trình soạn thảo văn bản thô nào trên Linux như gedit (đồ họa), nano (dòng lệnh đơn giản), vim hoặc vi (dòng lệnh nâng cao):

gedit firstscript

Bước 2: Cấp quyền thực thi

Sau khi được lưu, Linux vẫn coi tệp tin này chỉ là tệp văn bản thông thường. Muốn hệ điều hành chấp nhận chạy tệp tin này như một chương trình, bạn bắt buộc phải cấp quyền thực thi (Execute):

chmod 755 firstscript

Phân bổ quyền cụ thể:

  • Chủ sở hữu (Owner): Có quyền đọc, ghi và thực thi (rwx - 7).
  • Nhóm (Group): Chỉ có quyền đọc và thực thi (r-x - 5).
  • Người khác (Others): Chỉ có quyền đọc và thực thi (r-x - 5).

Bước 3: Thực thi script

Có 3 cách chạy một tệp tin Shell Script từ Terminal:

  • Cách 1: bash firstscript
  • Cách 2: sh firstscript
  • Cách 3: ./firstscript (Chỉ chạy được khi file đã được gán quyền thực thi bằng chmod).

Ví dụ script đầu tiên:

#!/bin/bash
clear
echo "This is my first script!!!"

Giải thích ý nghĩa từng dòng lệnh:

  • #!/bin/bash: Dòng này được gọi là Shebang. Nó cho nhân hệ điều hành biết chương trình thông dịch nào sẽ dùng để chạy script này (ở đây là Bash Shell).
  • clear: Xóa sạch toàn bộ nội dung hiển thị hiện tại trên cửa sổ Terminal.
  • echo "This is my first script!!!": In chuỗi văn bản ra màn hình.

2. Tại sao chạy script lại báo "Permission denied"?

Khi bạn cố gắng thực thi script bằng cú pháp ./firstscript và nhận được thông báo lỗi:

bash: ./firstscript: Permission denied

Nguyên nhân: Do tệp tin vừa tạo mới chưa được cấp quyền thực thi (Execute) cho người dùng hiện hành.

Cách khắc phục: Chạy lệnh gán quyền sau trước khi chạy lại script:

chmod 755 firstscript

3. Shell tìm script ở đâu?

Để Shell tìm thấy và chạy đúng script của bạn, có 2 cách thực hiện:

  1. Cách 1: Sử dụng đường dẫn tuyệt đối (Absolute Path)

    Trỏ thẳng vị trí lưu trữ từ gốc / của hệ thống:

    /root/shellscript/hello

    Hoặc chạy thông qua trình thông dịch:

    bash /root/shellscript/hello

    Cách này giúp bạn chạy được file dù đang làm việc ở bất cứ thư mục nào.

  2. Cách 2: Đăng ký thư mục chứa script vào biến PATH

    Khai báo đường dẫn thư mục vào danh sách tìm kiếm lệnh tự động của Shell:

    PATH=$PATH:/root/shellscript

    Sau khi cập nhật PATH, bạn có thể chạy script ở bất kỳ đâu chỉ bằng cách gõ tên file:

    hello

4. Bài tập thực hành đầu tiên

Hãy viết và chạy thử đoạn script sau:

clear
echo "Hello $USER"
echo -e "Today is \c"; date
echo -e "Number of user login : \c"; who | wc -l
echo "Calendar"
cal
exit 0

Giải thích chi tiết các thành phần trong đoạn code:

  • $USER: Biến hệ thống chứa tên của người dùng đang đăng nhập (ví dụ: bang).
  • date: Lệnh hiển thị ngày giờ hiện hành của hệ thống.
  • who | wc -l: Sử dụng cơ chế đường ống (Pipe). Lệnh who liệt kê danh sách người dùng đang đăng nhập, kết quả được gửi làm đầu vào cho lệnh wc -l để đếm tổng số dòng (tương ứng với số người dùng đang online).
  • cal: Hiển thị lịch của tháng hiện hành.
  • exit 0: Kết thúc chương trình và trả về mã trạng thái 0 (thành công) cho hệ thống.

5. Biến trong Shell

Biến (Variable) là các vùng nhớ dùng để lưu trữ dữ liệu tạm thời trong bộ nhớ RAM khi chạy script.

Ví dụ gán giá trị cho biến:

name=Bang
age=20

Có hai loại biến chính trong Shell:

  1. Biến hệ thống (System Variables):

    Là các biến được tạo và quản lý sẵn bởi hệ điều hành Linux. Tên của chúng thường được quy định viết bằng chữ HOA.

    Biến Ý nghĩa
    HOME Thư mục home cá nhân của người dùng hiện hành.
    PATH Danh sách các thư mục tìm kiếm lệnh của Shell.
    PWD Đường dẫn thư mục làm việc hiện tại.
    SHELL Đường dẫn chương trình Shell đang sử dụng.
    LOGNAME Tên tài khoản người dùng đăng nhập.
    BASH_VERSION Phiên bản của Bash Shell đang chạy.

    Ví dụ in giá trị biến hệ thống:

    echo $PWD
    /home/bang
  2. Biến do người dùng tự định nghĩa (User Defined Variables):

    Do lập trình viên tự khai báo để phục vụ mục đích lưu trữ dữ liệu cá nhân trong script. Viết bằng chữ thường để phân biệt.

    n=10
name=Bang
score=100

6. Quy tắc đặt tên biến

Khi tự đặt tên biến trong Shell, bạn phải tuân thủ nghiêm ngặt các quy tắc sau:

  • Bắt buộc phải bắt đầu bằng một chữ cái (a-z, A-Z) hoặc dấu gạch dưới _.
  • Ký tự tiếp theo có thể là chữ cái, số (0-9) hoặc dấu gạch dưới _.
  • Không được chứa các ký tự đặc biệt (như -, @, #, $...).

Ví dụ đặt tên hợp lệ: d100, a1_1, _head, _abc_.

Ví dụ đặt tên SAI:

  • 1name=Bang (Sai vì bắt đầu bằng số).
  • my-name=Bang (Sai vì chứa ký tự đặc biệt là dấu gạch ngang).

7. Lưu ý cực kỳ quan trọng khi khai báo biến

⚠️ Quy tắc khoảng trắng: Tuyệt đối không được phép chèn khoảng trắng xung quanh dấu bằng = khi gán giá trị biến.

Ví dụ gán SAI:

n = 10
n= 10
n =10

Ví dụ gán ĐÚNG:

n=10

Phân biệt chữ hoa và chữ thường:

Shell phân biệt chữ hoa và chữ thường (case-sensitive). Hai biến sau là hoàn toàn độc lập và lưu hai giá trị khác nhau:

No=10
no=20

Định nghĩa biến rỗng:

name=

Hoặc:

name=""

In giá trị của biến:

Bắt buộc phải sử dụng ký tự đô la $ đứng trước tên biến để lấy giá trị của nó ra:

echo $name

Bạn có thể nhúng trực tiếp biến vào trong chuỗi đặt trong cặp dấu ngoặc kép:

echo "My name is $LOGNAME"

8. Tìm lỗi trong đoạn code mẫu

Xét đoạn mã nguồn Shell chứa lỗi sau:

myname=Vivek
myos = TroubleOS
myno=5

echo "My name is $myname"
echo "My os is $myos"
echo "My number is myno, can you see this number"

Các lỗi được phát hiện và cách sửa đổi:

  1. Lỗi 1 tại dòng 2: myos = TroubleOS
    Có khoảng trắng xung quanh dấu = khiến Shell hiểu lầm myos là một tên lệnh và dấu bằng là đối số.
    Sửa lại: myos=TroubleOS
  2. Lỗi 2 tại dòng 6: echo "My number is myno"
    In ra trực tiếp chữ "myno" thay vì giá trị của biến vì thiếu ký tự $ đứng trước.
    Sửa lại: echo "My number is $myno"

9. Lệnh echo

Lệnh echo dùng để hiển thị các chuỗi ký tự ra màn hình.

Cú pháp:

echo [options] "nội_dung"

Option -n (Không xuống dòng):

Lệnh echo mặc định sẽ tự động thêm ký tự xuống dòng ở cuối chuỗi. Option -n giúp tắt tính năng này để lệnh echo tiếp theo in nối tiếp trên cùng 1 dòng.

echo -n "Hello "
echo "World"
Hello World

Option -e (Kích hoạt ký tự thoát - Escape characters):

Cho phép diễn giải các ký tự điều khiển định dạng văn bản đặc biệt:

Ký tự Ý nghĩa định dạng
\n Ngắt dòng và xuống dòng mới (newline).
\t Chèn một khoảng tab ngang (horizontal tab).
\a Phát ra một tiếng beep cảnh báo hệ thống (alert).
\b Xóa ngược lại 1 ký tự phía trước (backspace).
\\ In ra duy nhất một ký tự gạch chéo ngược \.

10. Tính toán bằng expr

Mặc định Shell xem mọi giá trị của biến là chuỗi ký tự. Để thực hiện các phép toán số học cơ bản, ta phải sử dụng công cụ hỗ trợ tính toán expr (Expression).

Cú pháp cơ bản:

expr toán_hạng1 toán_tử toán_hạng2

Ví dụ các phép toán:

expr 1 + 3
expr 5 - 2
expr 10 / 2
expr 20 % 3
expr 10 \* 3

⚠️ Quy tắc sử dụng expr bắt buộc:
1. Phải có khoảng trắng phân cách giữa các toán hạng và toán tử. Ví dụ expr 5 + 3 là ĐÚNG, expr 5+3 là SAI.
2. Phải thêm dấu gạch chéo ngược \ đứng trước dấu nhân * (tức là \*) để ngăn Shell diễn giải nó như ký tự đại diện wildcard.

Sử dụng expr gán giá trị hoặc in kết quả lồng nhau:

Sử dụng cặp dấu nháy ngược ` (backquote - phím nằm dưới phím Esc) để lấy kết quả tính toán của lệnh expr nhúng vào chuỗi:

echo "Tong la `expr 6 + 3`"
Tong la 9

11. Exit Status (Mã trạng thái kết thúc)

Khi bất kỳ câu lệnh nào trong hệ điều hành Linux chạy xong, nó luôn trả về cho hệ thống một mã số nguyên nằm trong khoảng từ 0 đến 255 để biểu diễn trạng thái kết thúc.

  • Mã bằng 0: Lệnh thực thi thành công hoàn toàn (Success).
  • Mã khác 0 (1-255): Lệnh gặp lỗi trong quá trình thực thi (Failure/Error).

Để xem mã trạng thái kết thúc của câu lệnh vừa chạy gần nhất, sử dụng biến đặc biệt $?:

echo $?

Ví dụ lệnh thành công:

date
echo $?
0

Ví dụ lệnh lỗi (xóa file không tồn tại):

rm file_khong_ton_tai
echo $?
1 (hoặc mã khác 0 tùy lỗi)

12. Đọc dữ liệu từ bàn phím

Lệnh read dừng chương trình để chờ đợi và nhận dữ liệu nhập vào từ bàn phím của người dùng, sau đó gán dữ liệu đó vào biến.

Ví dụ script nhận tên:

echo "Your first name please:"
read fname
echo "Hello $fname"

Nếu người dùng nhập vào chữ Bang và gõ Enter, kết quả in ra sẽ là:

Hello Bang

13. Thực hiện nhiều lệnh trên một dòng

Bạn có thể viết nhiều câu lệnh nằm liên tiếp nhau trên cùng một dòng lệnh duy nhất bằng cách phân tách chúng bởi dấu chấm phẩy ;.

echo Hello ; date

14. Truyền tham số cho Shell Script (Arguments)

Khi gọi chạy một script, ta có thể truyền thêm các đối số trực tiếp từ dòng lệnh:

./myscript foo bar

Hệ điều hành sẽ tự động gán các đối số này vào các biến số tương ứng dựa trên vị trí của chúng:

  • $0: Tên của tệp tin script đang chạy (ở đây là ./myscript).
  • $1: Tham số truyền vào ở vị trí đầu tiên (foo).
  • $2: Tham số truyền vào ở vị trí thứ hai (bar).
  • $3, $4...: Các tham số tiếp theo.

Các biến đối số đặc biệt quan trọng:

Biến Ý nghĩa
$# Trả về tổng số lượng tham số được truyền vào script (không tính $0).
$* Lấy tất cả các tham số truyền vào dưới dạng một chuỗi đơn duy nhất.
$@ Lấy tất cả các tham số truyền vào, phân tách thành các chuỗi riêng biệt.

15. Hướng dẫn giải chi tiết các bài tập cuối buổi thực hành

Dưới đây là mã nguồn Shell Script hoàn chỉnh và hướng dẫn kiểm thử cho các bài tập thực hành buổi 2:

Bài tập 1: Thực hiện lại toàn bộ ví dụ trong tài liệu

Yêu cầu: Tạo lại và chạy thử các tập tin script ví dụ về gán biến, lệnh echo, toán tử expr, tham số $1, $2.

Bài tập 2: Viết script hiển thị thông tin hệ thống và chờ nhấn phím

Tạo file script đặt tên là sysinfo.sh:

#!/bin/bash
clear
echo "==== THÔNG TIN HỆ THỐNG ===="
echo "Ngày giờ hiện tại : $(date)"
echo "Tên người dùng    : $USER"
echo "Thư mục hiện hành : $PWD"
echo "Thư mục cá nhân   : $HOME"
echo "============================"
echo -n "Nhấn phím Enter để tiếp tục..."
read dummy
exit 0

Bài tập 3: Viết script tính toán số học từ 2 tham số truyền vào

Tạo file script đặt tên là calculator.sh. Có kiểm tra điều kiện đầu vào:

#!/bin/bash
# Kiểm tra xem người dùng có truyền đủ 2 tham số hay không
if [ $# -lt 2 ]; then
  echo "Lỗi: Vui lòng truyền vào 2 số làm tham số dòng lệnh!"
  echo "Ví dụ: ./calculator.sh 15 5"
  exit 1
fi

num1=$1
num2=$2

echo "==== KẾT QUẢ TÍNH TOÁN ===="
echo "$num1 + $num2 = `expr $num1 + $num2`"
echo "$num1 - $num2 = `expr $num1 - $num2`"
echo "$num1 * $num2 = `expr $num1 \* $num2`"

# Kiểm tra phép chia cho số 0
if [ $num2 -eq 0 ]; then
  echo "Lỗi: Số chia bằng 0, không thể thực hiện phép chia và chia lấy dư."
else
  echo "$num1 / $num2 = `expr $num1 / $num2`"
  echo "$num1 % $num2 = `expr $num1 % $num2`"
fi
echo "==========================="
exit 0

Bài tập 4: Viết script taothumuc và xử lý lỗi

Tạo file script đặt tên là taothumuc.sh. Sử dụng mã exit status để bắt lỗi:

#!/bin/bash
echo -n "Nhập tên thư mục muốn tạo: "
read folder_name

# Tạo thư mục và chuyển hướng đầu ra lỗi stderr sang /dev/null để ẩn thông báo lỗi mặc định
mkdir "$folder_name" 2>/dev/null

# Kiểm tra mã trạng thái kết thúc của lệnh mkdir
if [ $? -ne 0 ]; then
  echo "Khong the tao duoc thu muc!!!"
  exit 1
else
  echo "Thư mục '$folder_name' đã được tạo thành công!"
  exit 0
fi

Bài tập 5: Viết script thuchien thực thi câu lệnh truyền vào

Tạo file script đặt tên là thuchien.sh. Sử dụng biến $*:

#!/bin/bash
if [ $# -lt 1 ]; then
  echo "Lỗi: Vui lòng cung cấp câu lệnh muốn thực thi!"
  echo "Ví dụ: ./thuchien.sh mkdir test_dir"
  exit 1
fi

echo "Đang thực hiện lệnh: $*"
echo "-----------------------------------"
# Thực thi toàn bộ chuỗi đối số như một câu lệnh thực tế
$*
exit 0
