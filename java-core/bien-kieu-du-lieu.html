2.1 Primitive Data Types

Khái niệm và mục đích sử dụng

Primitive Data Types (Kiểu dữ liệu nguyên thủy) là các kiểu dữ liệu cơ bản được Java cung cấp sẵn để lưu trữ giá trị đơn giản như số nguyên, số thực, ký tự hoặc giá trị đúng/sai.

Java có tổng cộng 8 kiểu dữ liệu nguyên thủy:

Kiểu dữ liệu Kích thước Phạm vi giá trị
byte1 byte-128 đến 127
short2 bytes-32,768 đến 32,767
int4 bytes$-2^{31}$ đến $2^{31}-1$
long8 bytes$-2^{63}$ đến $2^{63}-1$
float4 bytesSố thực đơn chính xác (độ chính xác đơn)
double8 bytesSố thực kép chính xác (độ chính xác kép)
char2 bytesKý tự đơn trong bảng mã Unicode
booleanPhụ thuộc JVMtrue hoặc false

Mục đích sử dụng:

  • Tiết kiệm tối đa bộ nhớ RAM
  • Tăng tốc độ tính toán thô của CPU (hiệu năng xử lý cao)
  • Thực hiện các phép toán học cơ bản trực tiếp
Cơ chế hoạt động bên trong

Các biến thuộc kiểu nguyên thủy (Primitive variables) lưu trực tiếp giá trị thực tế của chúng bên trong các ô nhớ thuộc vùng nhớ Stack.

Ví dụ: int age = 20;

Bộ nhớ STACK [Biến age] ┌────────┐ │ 20 │ └────────┘ (Không trỏ địa chỉ bộ nhớ hay tạo đối tượng ở Heap)
Cú pháp và quy tắc
datatype variableName = value;

Ví dụ khai báo:

int age = 20;
double salary = 1500.5;
char grade = 'A';
boolean isActive = true;

Lưu ý bắt buộc khi viết code:

  • Kiểu long cần hậu tố L hoặc l phía sau giá trị: long num = 9999999999L;
  • Kiểu float cần hậu tố F hoặc f: float rate = 5.75F;
Ví dụ mã nguồn đơn giản
public class Main {
    public static void main(String[] args) {
        int age = 20;
        double gpa = 3.5;
        char grade = 'A';
        boolean passed = true;

        System.out.println(age);
        System.out.println(gpa);
        System.out.println(grade);
        System.out.println(passed);
    }
}
Ví dụ thực tế trong dự án

Khai báo lưu trữ thông tin thực tế trong các hệ thống phần mềm doanh nghiệp:

int userId = 1001;
double accountBalance = 15000000.0;
boolean isVerified = true;
char gender = 'M';

2.2 Reference Data Types

Khái niệm và mục đích sử dụng

Reference Data Types (Kiểu dữ liệu tham chiếu) là các kiểu dữ liệu lưu trữ địa chỉ bộ nhớ (con trỏ tham chiếu) trỏ đến một thực thể đối tượng thực tế nằm trên bộ nhớ Heap thay vì lưu trữ giá trị dữ liệu thô trực tiếp.

Các kiểu dữ liệu tham chiếu phổ biến:

  • String (Chuỗi ký tự)
  • Mảng (Array)
  • Các lớp tự định nghĩa (Class)
  • Interface
  • Enum
Cơ chế hoạt động bên trong

Ví dụ: String name = "Bang";

Vùng nhớ STACK Vùng nhớ HEAP ┌──────────────┐ ┌─────────────────────┐ │ name: 0x7a12 ├────────────────────>│ Address 0x7a12: │ └──────────────┘ │ "Bang" (String Obj) │ └─────────────────────┘

Biến name nằm ở Stack chỉ chứa giá trị là địa chỉ bộ nhớ 0x7a12. Khi gọi biến này, JVM tự động đi tìm theo địa chỉ ở Heap để đọc dữ liệu chuỗi thực tế.

Cú pháp và quy tắc
String name = "Bang";
Person person = new Person();
int[] numbers = {1, 2, 3};

Giá trị mặc định: Khác với kiểu nguyên thủy có giá trị mặc định bằng số hoặc false, giá trị mặc định của mọi biến kiểu tham chiếu khi không được gán khởi tạo là: null.

Ví dụ mã nguồn đơn giản
public class Main {
    public static void main(String[] args) {
        String name = "Bang";
        System.out.println(name);
        System.out.println(name.length()); // Gọi được phương thức đi kèm của Object
    }
}
Ví dụ thực tế trong dự án

Xử lý dữ liệu nghiệp vụ thực tế trong Spring Boot:

User user = new User();
List<User> users = new ArrayList<>();

// Tìm kiếm người dùng, nếu không có thì trả về tham chiếu null
User userDb = userRepository.findById(id).orElse(null);

2.3 Khai báo biến

Khái niệm và mục đích sử dụng

Biến là tên định danh đại diện cho một vùng nhớ bộ nhớ được hệ thống cấp phát để lưu trữ dữ liệu thay đổi trong suốt quá trình chạy chương trình.

Cơ chế hoạt động bên trong

Khi thực hiện khai báo: int age = 20;

JVM cấp phát ô nhớ 4 bytes tại Stack và gán giá trị nhị phân tương ứng với 20. Khi thực hiện ghi đè giá trị mới: age = 25;, giá trị 20 cũ bị ghi đè hoàn toàn và ô nhớ chứa giá trị mới là 25.

Cú pháp và quy tắc

1. Chỉ khai báo chưa khởi tạo:

int age;

2. Khai báo kèm khởi tạo giá trị:

int age = 20;

3. Khai báo nhiều biến cùng kiểu dữ liệu trên một dòng:

int a = 1, b = 2, c = 3;

Quy tắc đặt tên biến trong Java:

  • Không bắt đầu đặt tên bằng số (Ví dụ sai: 1age).
  • Không chứa các ký tự đặc biệt (Ví dụ sai: @student, my-name).
  • Sử dụng quy chuẩn đặt tên dạng camelCase (lạc đà) bắt đầu bằng chữ thường (Ví dụ đúng: studentAge, totalPrice, isLogin).
  • Tên phải mang ý nghĩa rõ ràng, mô tả đúng dữ liệu lưu giữ.
Ví dụ mã nguồn đơn giản
public class Main {
    public static void main(String[] args) {
        String fullName = "Tran Vu Bang";
        int age = 20;

        System.out.println(fullName);
        System.out.println(age);
    }
}
Ví dụ thực tế trong dự án

Thiết lập biến lưu trữ thông tin đăng nhập trong các form xử lý:

String email = "bang@gmail.com";
String password = "xxxx";
boolean rememberMe = true;

2.4 Type Casting (Ép kiểu dữ liệu)

Khái niệm và mục đích sử dụng

Type Casting là quá trình chuyển đổi giá trị của một biến từ kiểu dữ liệu này sang một kiểu dữ liệu khác tương thích.

Cơ chế hoạt động bên trong

1. Ép kiểu tự động (Implicit / Widening Casting)

Xảy ra khi chuyển từ kiểu dữ liệu có kích thước nhỏ sang kiểu lớn hơn:

byte ──> short ──> int ──> long ──> float ──> double

JVM tự động thực thi chạy ngầm vì không có nguy cơ mất mát dữ liệu.

2. Ép kiểu tường minh (Explicit / Narrowing Casting)

Xảy ra khi chuyển từ kiểu dữ liệu có kích thước lớn về kiểu nhỏ hơn:

double ──> float ──> long ──> int ──> short ──> byte

Lập trình viên phải ra lệnh tường minh. Quá trình này có thể dẫn tới tràn số hoặc mất phần thập phân sau số thực.

Cú pháp và quy tắc

Cú pháp ép kiểu tự động:

int a = 10;
double b = a; // Tự động nhận giá trị b = 10.0

Cú pháp ép kiểu tường minh:

double x = 9.8;
int y = (int) x; // Trích xuất và chỉ giữ phần nguyên y = 9
Ví dụ mã nguồn đơn giản
public class Main {
    public static void main(String[] args) {
        int age = 20;
        double ageDouble = age;
        System.out.println(ageDouble); // 20.0

        double salary = 9999.99;
        int salaryInt = (int) salary;
        System.out.println(salaryInt); // 9999 (bị mất 0.99)
    }
}
Ví dụ thực tế trong dự án

Lấy giá trị thực tế sau khi tính toán tiền tệ và làm tròn sang số nguyên để phân trang hoặc in hóa đơn:

double totalAmount = 99999.99;
int roundedAmount = (int) totalAmount; // roundedAmount = 99999

2.5 Wrapper Classes

Khái niệm và mục đích sử dụng

Wrapper Class là lớp đối tượng được cung cấp để đóng gói (bao bọc) các kiểu dữ liệu nguyên thủy thô tương ứng thành các thực thể Object hoàn chỉnh.

Bảng đối chiếu kiểu nguyên thủy và Wrapper Class:

  • byte → Byte
  • short → Short
  • int → Integer
  • long → Long
  • float → Float
  • double → Double
  • char → Character
  • boolean → Boolean

Mục đích sử dụng:

  • Bắt buộc khi sử dụng trong Collection Framework (chỉ nhận đối tượng)
  • Dùng trong khai báo Generic kiểu mẫu
  • Cung cấp nhiều hàm tĩnh tiện ích đi kèm (ví dụ: chuyển đổi kiểu chữ sang số)
Cơ chế hoạt động bên trong

Biến kiểu nguyên thủy được cấp phát trực tiếp tại Stack. Biến kiểu Wrapper do bản chất là thực thể lớp (Object) nên giá trị thực tế được lưu trên Heap, còn Stack chỉ giữ địa chỉ trỏ sang Heap tương tự kiểu tham chiếu.

Cú pháp và quy tắc
Integer age = 20;
Double salary = 1000.5;
Boolean active = true;
Ví dụ mã nguồn đơn giản
public class Main {
    public static void main(String[] args) {
        Integer number = 100;
        System.out.println(number.doubleValue()); // Chuyển sang dạng double: 100.0
        System.out.println(number.toString());    // Chuyển sang dạng chuỗi: "100"
    }
}
Ví dụ thực tế trong dự án

Khai báo cấu trúc danh sách trong dự án (Collection chỉ chấp nhận đối tượng Wrapper):

// LỖI BIÊN DỊCH: List<int> numbers = new ArrayList<>();
List<Integer> numbers = new ArrayList<>(); // Khai báo chuẩn xác

2.6 Autoboxing và Unboxing

Khái niệm và mục đích sử dụng
  • Autoboxing: Tiến trình Java tự động chuyển đổi từ kiểu dữ liệu nguyên thủy sang đối tượng Wrapper tương ứng.
  • Unboxing: Tiến trình Java tự động chuyển ngược từ thực thể đối tượng Wrapper về kiểu dữ liệu nguyên thủy thô.
Cơ chế hoạt động bên trong

Trình biên dịch Java tự động dịch chuyển đổi code khi build chương trình chạy ngầm:

1. Khi viết code: Integer number = 10; (Autoboxing)

Trình biên dịch dịch thành: Integer number = Integer.valueOf(10);

2. Khi viết code: int x = number; (Unboxing)

Trình biên dịch dịch thành: int x = number.intValue();

Cú pháp và quy tắc
Integer x = 10; // Autoboxing
int y = x;      // Unboxing
Ví dụ mã nguồn đơn giản
public class Main {
    public static void main(String[] args) {
        Integer age = 20; // Autoboxing
        int value = age;  // Unboxing
        System.out.println(value);
    }
}
Ví dụ thực tế trong dự án

Cơ chế tự động chuyển đổi giúp lập trình viên thao tác cực kỳ tự nhiên khi nạp dữ liệu thô vào Collection:

List<Integer> scores = new ArrayList<>();

scores.add(10); // Thực tế JVM chạy: scores.add(Integer.valueOf(10));
scores.add(20);

int score = scores.get(0); // Thực tế JVM chạy: scores.get(0).intValue();

Nhờ cơ chế này, mã nguồn dự án của bạn sạch sẽ hơn rất nhiều vì không cần gọi ép kiểu thủ công liên tục.
