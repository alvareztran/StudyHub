9.1 String Pool

Khái niệm và mục đích sử dụng

String Pool (Hồ chứa chuỗi) là một phân vùng bộ nhớ đặc biệt nằm bên trong bộ nhớ Heap của JVM. String Pool được thiết kế để lưu trữ và tái sử dụng các chuỗi ký tự giống nhau nhằm tối ưu bộ nhớ RAM, tránh tạo thừa các đối tượng trùng lắp.

Cơ chế hoạt động bên trong

1. Trường hợp khởi tạo bằng String Literal (Khai báo trực tiếp)

String s1 = "Java";
String s2 = "Java";

JVM kiểm tra trong String Pool trước. Nếu chuỗi "Java" chưa tồn tại, nó sẽ tạo mới đối tượng này trong Pool. Khi khai báo s2 = "Java", JVM phát hiện chuỗi đã có sẵn nên chỉ việc trỏ tham chiếu của s2 về chung địa chỉ ô nhớ đó.

Vùng nhớ STACK Vùng nhớ HEAP (String Pool) ┌──────────────┐ ┌────────────────────────┐ │ s1: 0x101 ├─────────────────>│ Address 0x101: "Java" │ ├──────────────┤ │ │ │ s2: 0x101 ├─────────────────>│ (Dùng chung ô nhớ) │ └──────────────┘ └────────────────────────┘

Khi đó phép so sánh địa chỉ: s1 == s2 trả về kết quả là true.

2. Trường hợp khởi tạo bằng từ khóa new

String s1 = new String("Java");
String s2 = new String("Java");

JVM luôn bỏ qua String Pool để tạo ra hai đối tượng độc lập hoàn toàn nằm trên vùng nhớ Heap thô thông thường.

Vùng nhớ STACK Vùng nhớ HEAP thô ┌──────────────┐ ┌───────────────────────────────┐ │ s1: 0x900a ├─────────────────>│ Address 0x900a: String Object │ ├──────────────┤ ├───────────────────────────────┤ │ s2: 0x900b ├─────────────────>│ Address 0x900b: String Object │ └──────────────┘ └───────────────────────────────┘

Khi đó phép so sánh địa chỉ: s1 == s2 trả về kết quả là false.

Cú pháp và quy tắc

Quy tắc lập trình hiệu năng tốt: Luôn ưu tiên dùng String Literal:

String name = "Bang";

Tuyệt đối hạn chế khởi tạo qua new trừ trường hợp bắt buộc:

String name = new String("Bang"); // Tạo thêm 1 đối tượng dư thừa trên Heap thô
Ví dụ mã nguồn đơn giản
public class Main {
    public static void main(String[] args) {
        String a = "Hello";
        String b = "Hello";

        System.out.println(a == b);      // true (cùng tham chiếu String Pool)
        System.out.println(a.equals(b)); // true (so sánh nội dung text bên trong)
    }
}
Ví dụ thực tế trong dự án

Các hằng số vai trò (Role) hoặc cấu hình hệ thống cố định lặp lại liên tục trong mã nguồn sẽ được JVM gom giữ tập trung trong Pool để hạn chế rác bộ nhớ:

String role = "ADMIN";

9.2 Immutable Object (Tính bất biến của String)

Khái niệm và mục đích sử dụng

Immutable Object là đối tượng có trạng thái dữ liệu bên trong **không thể bị sửa đổi** sau khi được khởi sinh thành công. Lớp String trong Java là một Immutable Class điển hình.

Mục đích:

  • Giúp chia sẻ chuỗi an toàn tuyệt đối trong môi trường đa luồng (Thread-safe) mà không cần khóa đồng bộ
  • Bảo mật hệ thống (các tham số nhạy cảm như URL, User, Pass dạng String sẽ không bị thay đổi ngầm)
  • Giúp cơ chế lưu String Pool hoạt động tin cậy (nếu sửa được s1 thì s2 dùng chung sẽ bị ảnh hưởng lỗi)
Cơ chế hoạt động bên trong

Khi thực hiện biến đổi chuỗi: String name = "Java"; name = name.concat(" Core");

JVM hoàn toàn không sửa đổi chuỗi "Java" ban đầu. Thay vào đó, nó cấp phát một ô nhớ mới chứa nội dung ghép "Java Core" rồi chuyển con trỏ name trỏ sang ô nhớ mới đó. Chuỗi "Java" cũ vẫn tồn tại nguyên vẹn trong bộ nhớ.

Cú pháp và quy tắc

Sai lầm kinh điển của lập trình viên mới học:

String name = "Bang";
name.toUpperCase(); // Chuyển chữ hoa nhưng không gán lại biến
System.out.println(name); // Kết quả vẫn in ra chữ thường: "Bang"

Để ghi nhận giá trị chuỗi sau biến đổi, bạn bắt buộc phải gán kết quả ngược lại biến:

name = name.toUpperCase(); // name trỏ sang chuỗi mới "BANG"
Ví dụ mã nguồn đơn giản
public class Main {
    public static void main(String[] args) {
        String text = "hello";
        String upper = text.toUpperCase();

        System.out.println(text);  // hello (chuỗi gốc bất biến)
        System.out.println(upper); // HELLO (chuỗi mới được sinh ra)
    }
}
Ví dụ thực tế trong dự án

Chuẩn hóa thông tin Email của người dùng gửi từ giao diện trước khi nạp vào Database:

String email = request.getEmail();
email = email.trim().toLowerCase(); // Bắt buộc phải gán lại kết quả

9.3 StringBuilder

Khái niệm và mục đích sử dụng

StringBuilder là lớp tiện ích hỗ trợ sửa đổi, thao tác xây dựng chuỗi động mà **không tạo ra bất kỳ đối tượng trung gian dư thừa nào** trong bộ nhớ Heap.

Mục đích: Khắc phục nhược điểm hiệu năng của tính bất biến String khi cần cộng chuỗi liên tục (ví dụ trong vòng lặp), giảm tải tối đa cho bộ dọn rác Garbage Collector.

Cơ chế hoạt động bên trong

Hãy phân tích tác hại của tính bất biến khi cộng chuỗi bằng toán tử + thông thường trong vòng lặp:

String text = "";
for (int i = 0; i < 1000; i++) {
    text += i; // Mỗi lượt chạy sinh ra 1 String Object tạm, tổng cộng 1000 objects!
}

Với StringBuilder, hệ thống cấp phát một mảng ký tự có kích thước động (resizable char array) chạy ngầm bên dưới. Mọi thao tác nối chữ chỉ ghi đè trực tiếp lên mảng ký tự chung đó, không tạo thêm bất kỳ đối tượng mới nào:

StringBuilder builder = new StringBuilder();
for (int i = 0; i < 1000; i++) {
    builder.append(i); // Chỉ có DUY NHẤT 1 object được thay đổi trạng thái liên tục
}
Cú pháp và quy tắc
StringBuilder sb = new StringBuilder();
sb.append("Text");
String result = sb.toString(); // Chuyển đổi về dạng String khi xử lý xong

Bảng phương thức tiện ích phổ biến của StringBuilder:

Phương thức Chức năng xử lý
append(value)Ghép nối thêm dữ liệu vào đuôi chuỗi hiện tại
insert(index, value)Chèn thêm dữ liệu vào vị trí chỉ số index định trước
delete(start, end)Xóa bỏ các ký tự từ vị trí start đến trước end
replace(start, end, str)Thay thế đoạn ký tự bằng chuỗi mới
reverse()Đảo ngược toàn bộ các ký tự trong chuỗi
Ví dụ mã nguồn đơn giản
public class Main {
    public static void main(String[] args) {
        StringBuilder sb = new StringBuilder();
        sb.append("Java");
        sb.append(" ");
        sb.append("Core");

        System.out.println(sb.toString()); // Java Core
    }
}
Ví dụ thực tế trong dự án

Thiết lập chuỗi truy vấn SQL động theo các bộ lọc tìm kiếm linh hoạt:

StringBuilder sql = new StringBuilder("SELECT * FROM users WHERE 1=1 ");

if (email != null) {
    sql.append("AND email = ? ");
}
if (status != null) {
    sql.append("AND status = ? ");
}

9.4 StringBuffer

Khái niệm và mục đích sử dụng

StringBuffer sở hữu đầy đủ tính năng thiết kế resizable char array tương tự như StringBuilder. Điểm khác biệt mấu chốt là mọi phương thức của StringBuffer đều được đồng bộ hóa luồng (Thread-safe).

Cơ chế hoạt động bên trong

Hầu hết các phương thức thao tác chuỗi trong mã nguồn JDK của StringBuffer được đánh dấu với từ khóa:

public synchronized StringBuffer append(String str)

Từ khóa synchronized ép buộc các luồng (threads) khác nhau khi truy cập vào cùng đối tượng StringBuffer phải xếp hàng tuần tự. Điều này giúp ngăn ngừa triệt để lỗi xung đột ghi đè dữ liệu đồng thời ở môi trường đa luồng (Multi-threading).

Ví dụ mã nguồn đơn giản
public class Main {
    public static void main(String[] args) {
        StringBuffer buffer = new StringBuffer("Hello");
        buffer.append(" ");
        buffer.append("Java");
        System.out.println(buffer.toString()); // Hello Java
    }
}
Ví dụ thực tế trong dự án

Thu thập và ghi log tập trung từ nhiều Thread nghiệp vụ xử lý song song đổ về chung một bộ đệm:

StringBuffer logBuffer = new StringBuffer();
logBuffer.append("Thread ID: ").append(Thread.currentThread().getId()).append(" - Task Done");

So sánh String, StringBuilder và StringBuffer

Tiêu chí so sánh String StringBuilder StringBuffer
Khả năng thay đổi Không (Immutable) Có (Mutable) Có (Mutable)
Độ an toàn đa luồng An toàn (Thread-safe) Không an toàn (Thread-unsafe) An toàn (Thread-safe)
Hiệu năng tốc độ Thấp (Do tạo nhiều object) Cao nhất (Không khóa đồng bộ) Trung bình - Cao (Do tốn chi phí khóa luồng)
Vị trí bộ nhớ Heap Trong String Pool (Literal) hoặc Heap thô Vùng nhớ Heap thô Vùng nhớ Heap thô
Nguyên tắc quyết định lựa chọn sử dụng
  • Sử dụng String khi: Chuỗi dữ liệu ít khi thay đổi, các biến cấu hình, khai báo vai trò (ví dụ: String role = "ADMIN") nhằm tận dụng tối đa String Pool giảm dung lượng bộ nhớ.
  • Sử dụng StringBuilder khi: Cần xử lý, ghép nối chuỗi liên tục với số lượng lớn trong các vòng lặp, tạo câu SQL động, định dạng HTML động trong phạm vi **đơn luồng** (Local variable) để đạt tốc độ xử lý nhanh tối đa.
  • Sử dụng StringBuffer khi: Gặp bài toán ghép chuỗi dùng chung được tác động ghi sửa đồng thời bởi **nhiều luồng chạy song song** cần đảm bảo tính toàn vẹn dữ liệu.
