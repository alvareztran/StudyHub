7.1 Class và Object

Khái niệm và mục đích sử dụng

Mô hình lập trình hướng đối tượng (OOP) tổ chức ứng dụng xoay quanh thực thể đời thực:

  • Class (Lớp): Bản thiết kế kỹ thuật (Blueprint) định nghĩa cấu trúc dữ liệu gồm các thuộc tính (fields) và các hành vi (methods) chung.
  • Object (Đối tượng): Là thực thể đại diện độc lập (Instance) được tạo ra từ Class, sở hữu ô nhớ vật lý trên Heap.
Cơ chế hoạt động bên trong

Khi gọi hàm khởi tạo: Student student = new Student();

JVM thực hiện quy trình sau:

  1. Cấp phát vùng nhớ trống trên Heap để chứa các biến thực thể (instance fields).
  2. Khởi tạo giá trị mặc định cho các fields (như 0 cho số, null cho kiểu đối tượng).
  3. Thực thi hàm khởi tạo (Constructor).
  4. Trả về địa chỉ ô nhớ Heap gán vào biến tham chiếu student đặt tại Stack.
Vùng nhớ STACK Vùng nhớ HEAP ┌─────────────────┐ ┌───────────────────────────────┐ │ student: 0x4422 ├─────────────────>│ Address 0x4422: Student Object│ └─────────────────┘ │ name = "Bang", age = 20 │ └───────────────────────────────┘
Cú pháp và quy tắc
class Student {
    String name;
    int age;
}

Khởi tạo đối tượng:

Student student = new Student();
Ví dụ mã nguồn đơn giản
class Student {
    String name;
    int age;
}

public class Main {
    public static void main(String[] args) {
        Student student = new Student();
        student.name = "Bang";
        student.age = 20;

        System.out.println(student.name); // "Bang"
        System.out.println(student.age);  // 20
    }
}
Ví dụ thực tế trong dự án

Thiết lập lớp thực thể (Entity) đại diện cho tài khoản người dùng Database:

class User {
    Long id;
    String email;
    String password;
}

// Tạo tài khoản mới:
User user = new User();

7.2 Constructor (Hàm khởi tạo)

Khái niệm và mục đích sử dụng

Constructor là một phương thức đặc biệt trùng tên lớp hoàn toàn, không có kiểu dữ liệu trả về, được gọi tự động duy nhất tại thời điểm đối tượng được khởi sinh bằng từ khóa new.

Mục đích: Khởi tạo các giá trị tham số cần thiết ngay khi tạo đối tượng, đảm bảo đối tượng không bị lỗi dữ liệu rỗng (inconsistent state).

Cơ chế hoạt động bên trong

Nếu bạn không khai báo bất kỳ constructor nào, trình biên dịch tự động chèn một **Constructor mặc định không tham số trống** (Default Constructor) lúc build code. Nếu bạn tự viết một Constructor có tham số, Default Constructor này sẽ tự biến mất.

Cú pháp và quy tắc
class User {
    // Tên trùng khớp class, không có kiểu trả về (kể cả void)
    User() {
        // Logic khởi tạo
    }
}
Ví dụ mã nguồn đơn giản
class Student {
    String name;

    Student(String name) {
        this.name = name; // Khởi tạo dữ liệu
    }
}

// Gọi khởi sinh:
Student student = new Student("Bang");
Ví dụ thực tế trong dự án
public User(String email, String password) {
    this.email = email;
    this.password = password; // Bắt buộc truyền email và pass khi tạo mới
}

7.3 this và super

Khái niệm và mục đích sử dụng
  • this: Con trỏ tự tham chiếu, đại diện cho chính đối tượng hiện tại đang hoạt động.
  • super: Con trỏ đại diện cho thực thể thuộc về lớp cha trực hệ gần nhất.
Cơ chế hoạt động bên trong

Lệnh this.name = name; giúp JVM phân biệt rõ thuộc tính name của đối tượng trên Heap (phía vế trái) với tham số đầu vào trùng tên name ở Stack Frame (phía vế phải).

Lệnh super() hoặc this() dùng trong constructor bắt buộc phải được đặt ở **dòng lệnh đầu tiên** của constructor đó.

Cú pháp và quy tắc

1. Cú pháp sử dụng this:

this.field;      // Gọi thuộc tính
this.method();   // Gọi phương thức đối tượng
this();          // Gọi constructor cùng lớp

2. Cú pháp sử dụng super:

super.field;     // Gọi thuộc tính của lớp cha
super.method();  // Gọi phương thức ghi đè của lớp cha
super();         // Gọi constructor của lớp cha
Ví dụ mã nguồn đơn giản
class Person {
    String name;
    Person(String name) {
        this.name = name;
    }
}

class Student extends Person {
    Student(String name) {
        super(name); // Gọi constructor cha Person
    }
}
Ví dụ thực tế trong dự án
public User(String email) {
    this.email = email; // Phân biệt thuộc tính và tham số cực kỳ rõ ràng
}

7.4 static

Khái niệm và mục đích sử dụng

Từ khóa static định nghĩa thuộc tính hoặc phương thức thuộc sở hữu của toàn bộ **Lớp (Class Level)** chứ không thuộc về bất kỳ đối tượng thực thể riêng lẻ nào. Tất cả thực thể sinh ra đều dùng chung một biến static duy nhất.

Cơ chế hoạt động bên trong

Biến static được khởi tạo duy nhất một lần khi Class Loader nạp Class vào JVM. Chúng được lưu trữ tại vùng nhớ đặc biệt dành riêng cho Class (thuộc Metaspace/Heap) và có vòng đời tồn tại vĩnh viễn suốt quá trình ứng dụng chạy.

Cú pháp và quy tắc
static datatype variableName;
static returnType methodName() { ... }

Quy tắc: Trong các phương thức tĩnh static method, bạn tuyệt đối không được phép gọi trực tiếp các biến hay phương thức phi tĩnh (non-static) của đối tượng vì lúc đó đối tượng có thể chưa được khởi sinh.

Ví dụ mã nguồn đơn giản
class Student {
    static String school = "Đại học Cần Thơ";
    String name;
}

public class Main {
    public static void main(String[] args) {
        // Truy cập trực tiếp qua tên Lớp, không cần "new"
        System.out.println(Student.school); 
    }
}
Ví dụ thực tế trong dự án

Khai báo hằng số hệ thống toàn cục dùng chung ở mọi nơi:

public static final String ROLE_ADMIN = "ADMIN";

7.5 final

Khái niệm và mục đích sử dụng

Từ khóa final dùng để ngăn chặn việc thay đổi dữ liệu hoặc cấu trúc kế thừa thiết kế trong Java.

Bảng phân tích ý nghĩa sử dụng tại từng vị trí:

Vị trí áp dụng Ý nghĩa ràng buộc
final variableBiến trở thành hằng số, không được phép gán lại giá trị mới sau khi khởi tạo.
final methodLớp con kế thừa không được phép ghi đè (Override) phương thức này.
final classKhóa hoàn toàn lớp, không cho phép bất kỳ lớp nào kế thừa lớp này nữa.
Ví dụ mã nguồn đơn giản
public class Main {
    public static void main(String[] args) {
        final int AGE_LIMIT = 18;
        // LỖI BIÊN DỊCH: AGE_LIMIT = 20;
    }
}
Ví dụ thực tế trong dự án

Thiết lập chuỗi khóa bí mật (Secret Key) dùng để mã hóa mã Token bảo mật JWT:

public static final String JWT_SECRET = "my_super_secret_key_123";

7.6 Encapsulation (Tính đóng gói)

Khái niệm và mục đích sử dụng

Đóng gói là kỹ thuật che giấu các thông tin/thuộc tính nhạy cảm bên trong của đối tượng khỏi truy cập trái phép từ bên ngoài và chỉ cho phép đọc, ghi thông qua các phương thức công khai.

Mục đích: Bảo vệ toàn vẹn dữ liệu, kiểm soát tính hợp lệ của dữ liệu trước khi nạp vào biến.

Cơ chế hoạt động bên trong

Bằng cách khóa thuộc tính bằng private, đối tượng bên ngoài không thể thay đổi giá trị bừa bãi. Hàm setter đóng vai trò là "chốt kiểm soát" dữ liệu đầu vào.

Cú pháp và quy tắc
private String name;

public String getName() {
    return name;
}

public void setName(String name) {
    if (name == null || name.trim().isEmpty()) {
        throw new IllegalArgumentException("Tên không hợp lệ!");
    }
    this.name = name;
}
Ví dụ mã nguồn đơn giản
class Student {
    private String name;

    public String getName() {
        return this.name;
    }
    public void setName(String name) {
        this.name = name;
    }
}
Ví dụ thực tế trong dự án

Bảo mật thuộc tính mật khẩu người dùng trong thực tế:

private String password; // Tuyệt đối không cho truy cập trực tiếp bằng user.password

7.7 Inheritance (Tính kế thừa)

Khái niệm và mục đích sử dụng

Kế thừa cho phép một lớp con (Subclass) tái sử dụng toàn bộ các thuộc tính và hành vi công khai của lớp cha (Superclass) giúp giảm thiểu trùng lặp code tối đa.

Cơ chế hoạt động bên trong

Khi đối tượng lớp con được tạo, vùng nhớ Heap của lớp con chứa cả các thuộc tính của cha. JVM tự động gọi Constructor của lớp cha trước (qua super() ngầm định) rồi mới thực thi Constructor của lớp con.

Ràng buộc Java: Java chỉ hỗ trợ **Đơn kế thừa lớp (Single Inheritance)** - nghĩa là một lớp con chỉ được phép kế thừa duy nhất từ 1 lớp cha trực tiếp. Java không hỗ trợ đa kế thừa lớp để tránh lỗi xung đột kim cương (Diamond Problem).

Cú pháp và quy tắc
class SubClass extends SuperClass {
    // Code bổ sung đặc trưng của lớp con
}
Ví dụ mã nguồn đơn giản
class Animal {
    void sound() {
        System.out.println("Animal Sound");
    }
}

class Dog extends Animal {
    // Dog tự động thừa hưởng phương thức sound()
}
Ví dụ thực tế trong dự án

Xây dựng hệ thống quản lý thông tin nhân sự trong công ty:

class Employee {
    String name;
    double baseSalary;
}

class Developer extends Employee {
    String programmingLanguage; // Chỉ dev mới có ngôn ngữ lập trình
}

7.8 Polymorphism (Tính đa hình)

Khái niệm và mục đích sử dụng

Đa hình là khả năng một tham chiếu kiểu cha đại diện cho nhiều đối tượng thực tế kiểu con khác nhau.

Cơ chế hoạt động bên trong

Khi ta gán: Animal animal = new Dog();

Trình biên dịch chỉ cho phép gọi các hàm khai báo ở kiểu cha Animal. Tuy nhiên, tại thời điểm chạy (Runtime), JVM sử dụng cơ chế **Dynamic Method Dispatch** để đi tìm và thực thi chính xác phương thức được ghi đè (Override) tại đối tượng con thực tế Dog.

Cú pháp và quy tắc
ParentType reference = new ChildType();
Ví dụ mã nguồn đơn giản
class Animal {
    void sound() { System.out.println("Animal"); }
}

class Dog extends Animal {
    @Override
    void sound() { System.out.println("Dog"); }
}

public class Main {
    public static void main(String[] args) {
        Animal animal = new Dog();
        animal.sound(); // JVM xuất ra: Dog (Đa hình Runtime)
    }
}
Ví dụ thực tế trong dự án

Khai báo linh hoạt cấu trúc danh sách trong thực tế phát triển phần mềm:

List<String> list = new ArrayList<>(); // List là Interface cha, ArrayList là con

7.9 Abstraction (Tính trừu tượng)

Khái niệm và mục đích sử dụng

Tính trừu tượng ẩn đi chi tiết cài đặt nghiệp vụ phức tạp ở bên dưới và chỉ trưng bày ra các giao diện/tính năng thiết kế tối giản cần thiết để tích hợp.

Cú pháp và quy tắc

Để triển khai tính trừu tượng trong Java, ta sử dụng hai công cụ chính:

  • Lớp trừu tượng (Abstract Class)
  • Giao diện (Interface)
Ví dụ mã nguồn đơn giản
abstract class Animal {
    abstract void sound(); // Phương thức trống không chứa mã nguồn xử lý
}
Ví dụ thực tế trong dự án

Thiết lập lớp cổng thanh toán dịch vụ trong dự án thương mại điện tử:

PaymentService paymentService;

Lớp Controller gọi thanh toán không cần quan tâm bên dưới hệ thống đang kết nối qua cổng Paypal, Stripe hay VNPay, giúp dễ dàng đổi nhà cung cấp mà không sửa đổi controller.

7.10 Interface

Khái niệm và mục đích sử dụng

Interface là bản cam kết (hợp đồng hành vi) buộc các lớp thực thi (implement) nó phải định nghĩa lại toàn bộ các phương thức trừu tượng.

Cú pháp và quy tắc
interface Flyable {
    void fly();
}

class Bird implements Flyable {
    @Override
    public void fly() {
        System.out.println("Bird is flying");
    }
}
Ví dụ thực tế trong dự án

Cơ chế thiết kế kiến trúc chuẩn hóa Spring Boot (tách biệt Interface Service và lớp triển khai thực tế):

public interface UserService {
    User findById(Long id);
}

@Service
public class UserServiceImpl implements UserService {
    @Override
    public User findById(Long id) {
        return userRepository.findById(id).orElseThrow();
    }
}

Cấu hình này giúp Spring tiêm phụ thuộc (Dependency Injection) linh hoạt thông qua Interface UserService, giảm thiểu tối đa liên kết cứng giữa các tầng nghiệp vụ hệ thống.
