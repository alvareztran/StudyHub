6.1 Array 1 chiều

Khái niệm và mục đích sử dụng

Mảng một chiều (One-Dimensional Array) là tập hợp chứa nhiều phần tử có cùng kiểu dữ liệu, được cấp phát liên tiếp nhau trong vùng nhớ Heap và truy xuất thông qua chỉ số (index).

Mục đích sử dụng:

  • Quản lý danh sách phần tử cùng loại thay vì khai báo hàng trăm biến đơn lẻ rời rạc
  • Tối ưu hóa không gian lưu trữ và truy cập nhanh
  • Làm nền tảng để tự xây dựng các cấu trúc dữ liệu phức tạp hơn như ArrayList, Stack, Queue
Cơ chế hoạt động bên trong

Mảng được JVM cấp phát một vùng nhớ liên tục (contiguous) trên bộ nhớ Heap.

Ví dụ: int[] numbers = {10, 20, 30, 40};

Vùng nhớ STACK Vùng nhớ HEAP ┌────────────────┐ ┌──────────────────────────┐ │ numbers: 0x5a ├───────────────>│ 0x5a: Index 0: 10 │ (Mỗi ô chiếm 4 bytes) └────────────────┘ │ Index 1: 20 │ │ Index 2: 30 │ │ Index 3: 40 │ └──────────────────────────┘

Công thức tính toán địa chỉ vật lý của phần tử ở chỉ số $i$:

$$\text{Địa chỉ phần tử } i = \text{Địa chỉ đầu (Base Address)} + i \times \text{Kích thước kiểu dữ liệu}$$

Nhờ phép toán số học cộng địa chỉ trực tiếp này, CPU có thể nhảy tới bất kỳ ô nhớ của index nào lập tức, giúp truy xuất mảng đạt hiệu năng tối đa với độ phức tạp thời gian: $O(1)$.

Cú pháp và quy tắc

1. Khai báo (Chưa cấp phát ô nhớ ở Heap):

int[] numbers; // Khuyên dùng cú pháp này

2. Khởi tạo kích thước cố định (Các phần tử nhận giá trị mặc định ban đầu là 0):

int[] numbers = new int[5]; // Mảng chứa tối đa 5 số nguyên

3. Khởi tạo và gán giá trị trực tiếp:

int[] numbers = {10, 20, 30, 40, 50};

Để lấy số lượng phần tử của mảng, dùng thuộc tính:

int length = numbers.length;
Ví dụ mã nguồn đơn giản
public class Main {
    public static void main(String[] args) {
        int[] numbers = {10, 20, 30, 40, 50};

        System.out.println(numbers[0]);      // 10
        System.out.println(numbers[4]);      // 50
        System.out.println(numbers.length);  // 5
    }
}
Ví dụ thực tế trong dự án

Lưu danh sách điểm số đánh giá sao của sản phẩm hoặc dãy nhiệt độ cảm biến:

double[] ratings = {4.5, 4.0, 5.0, 4.8};
double[] temperatures = {30.2, 30.5, 31.0, 29.8};

6.2 Array 2 chiều

Khái niệm và mục đích sử dụng

Mảng hai chiều thực chất là một mảng lớn mà mỗi phần tử bên trong nó lại chứa một mảng một chiều khác. Dùng để biểu diễn cấu trúc dạng bảng gồm Hàng (Rows) và Cột (Columns).

Cơ chế hoạt động bên trong

Trong Java, mảng hai chiều không phải là một ma trận phẳng liên tục thực tế, mà là **Mảng chứa các mảng (Array of Arrays)**.

matrix [Stack: 0x99] ↓ [Heap Array of References (0x99)] ┌────────────┬────────────┬────────────┐ │ Row 0: 0x01│ Row 1: 0x02│ Row 2: 0x03│ └─────┬──────┴─────┬──────┴─────┬──────┘ │ │ │ v v v [1, 2, 3] [4, 5, 6] [7, 8, 9] (Các mảng 1 chiều riêng biệt)

Mảng không đều (Jagged Array): Vì cơ chế này, các mảng con ở từng hàng hoàn toàn không bắt buộc phải có độ dài bằng nhau:

int[][] jaggedArray = {
    {1, 2},
    {3, 4, 5},
    {6}
};
Cú pháp và quy tắc

Khai báo và khởi tạo ma trận có kích thước cố định:

int[][] matrix = new int[3][4]; // 3 hàng, 4 cột

Khởi tạo trực tiếp:

int[][] matrix = {
    {1, 2, 3},
    {4, 5, 6},
    {7, 8, 9}
};
Ví dụ mã nguồn đơn giản
public class Main {
    public static void main(String[] args) {
        int[][] matrix = {
            {1, 2, 3},
            {4, 5, 6},
            {7, 8, 9}
        };
        System.out.println(matrix[1][2]); // In ra: 6 (Hàng chỉ số 1, cột chỉ số 2)
    }
}
Ví dụ thực tế trong dự án

Thiết lập bàn cờ vua, lưới tọa độ rạp phim, hoặc xử lý điểm ảnh Grayscale:

char[][] chessBoard = new char[8][8];
boolean[][] cinemaSeats = new boolean[10][20]; // true là ghế đã được đặt
int[][] imagePixels = new int[512][512];      // Ma trận điểm ảnh 512x512

6.3 Duyệt mảng

Khái niệm và mục đích sử dụng

Duyệt mảng (Array Traversal) là hành động truy xuất tuần tự qua tất cả các ô chỉ số của mảng để hiển thị dữ liệu, tìm kiếm phần tử, tính toán hoặc cập nhật dữ liệu.

Cú pháp và quy tắc

1. Sử dụng vòng lặp For truyền thống

Cho phép vừa đọc vừa ghi dữ liệu và tương tác trực tiếp với chỉ số index.

for (int i = 0; i < arr.length; i++) {
    // Xử lý với arr[i]
}

2. Sử dụng vòng lặp For-Each cải tiến (Enhanced For Loop)

Code ngắn gọn hơn. Tuy nhiên **chỉ hỗ trợ đọc dữ liệu** (Read-only), không cho phép cập nhật ghi đè phần tử và không có chỉ số index đi kèm.

for (int value : arr) {
    // Xử lý trực tiếp với giá trị value
}
Ví dụ mã nguồn đơn giản

1. Duyệt và in mảng 1 chiều:

int[] numbers = {10, 20, 30, 40};
for (int number : numbers) {
    System.out.println(number);
}

2. Duyệt qua mảng 2 chiều bằng 2 vòng lặp lồng nhau:

int[][] matrix = {
    {1, 2, 3},
    {4, 5, 6}
};
for (int i = 0; i < matrix.length; i++) {
    for (int j = 0; j < matrix[i].length; j++) {
        System.out.print(matrix[i][j] + " ");
    }
    System.out.println();
}
Ví dụ thực tế trong dự án

Tính tổng tiền giỏ hàng và quét tìm sản phẩm khớp mã ID trong danh sách:

// 1. Tính tổng hóa đơn
double[] prices = {100000.0, 250000.0, 50000.0};
double total = 0;
for (double price : prices) {
    total += price;
}

// 2. Tìm kiếm phần tử
Product foundProduct = null;
for (Product product : products) {
    if (product.getId() == targetId) {
        foundProduct = product;
        break; // Dừng duyệt ngay khi tìm thấy để tối ưu CPU
    }
}

6.4 Arrays Utility (Lớp tiện ích java.util.Arrays)

Khái niệm và mục đích sử dụng

Lớp tiện ích java.util.Arrays cung cấp một tập hợp các phương thức tĩnh (static methods) tối ưu hóa sẵn cấu trúc thuật toán để thao tác nhanh gọn với mảng thô.

Bảng phân tích cơ chế hoạt động các phương thức chính:

Phương thức Thuật toán cốt lõi bên dưới JVM Ứng dụng thực tế
Arrays.sort()Dual-Pivot Quicksort ($O(N \log N)$)Sắp xếp tăng dần
Arrays.binarySearch()Binary Search ($O(\log N)$)Tìm kiếm nhị phân cực nhanh
Arrays.equals()So sánh tuần tự từng ô nhớ phần tửKiểm tra tính trùng khớp dữ liệu
Arrays.fill()Vòng lặp ghi đè hàng loạtKhởi tạo giá trị mặc định ban đầu
Arrays.copyOf()System.arraycopy (Native copy cấp thấp)Sao chép nhân bản mảng
Cú pháp và quy tắc

Bắt buộc import thư viện trước khi gọi:

import java.util.Arrays;

Chú ý: Phương thức Arrays.binarySearch() chỉ hoạt động chính xác khi mảng truyền vào đã được **sắp xếp thứ tự tăng dần** từ trước.

Ví dụ mã nguồn đơn giản
import java.util.Arrays;

public class Main {
    public static void main(String[] args) {
        int[] numbers = {5, 2, 8, 1, 9};

        // 1. Sắp xếp mảng
        Arrays.sort(numbers);
        System.out.println(Arrays.toString(numbers)); // [1, 2, 5, 8, 9]

        // 2. Tìm kiếm nhị phân giá trị 8
        int index = Arrays.binarySearch(numbers, 8);
        System.out.println("Vị trí index: " + index); // 3

        // 3. Khởi tạo mảng toàn giá trị 100
        int[] arr = new int[3];
        Arrays.fill(arr, 100);
        System.out.println(Arrays.toString(arr)); // [100, 100, 100]
    }
}
Ví dụ thực tế trong dự án

Thực hiện sao lưu mảng dữ liệu trước khi chỉnh sửa và kiểm tra thay đổi:

// Sao lưu mảng
int[] backupData = Arrays.copyOf(sensorData, sensorData.length);

// Đối chiếu thay đổi cấu hình
if (Arrays.equals(oldConfigs, newConfigs)) {
    System.out.println("Không phát hiện thay đổi cấu hình mới nào");
}
