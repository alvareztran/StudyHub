3.1 Arithmetic Operators (Toán tử số học)

Khái niệm và mục đích sử dụng

Toán tử số học được sử dụng để thực hiện các phép tính toán học cơ bản như cộng, trừ, nhân, chia và chia lấy dư.

Các toán tử số học trong Java:

Toán tử Ý nghĩa
+Phép toán Cộng
-Phép toán Trừ
*Phép toán Nhân
/Phép toán Chia
%Phép toán Chia lấy dư (Modulo)
++Tăng giá trị lên 1 đơn vị
--Giảm giá trị đi 1 đơn vị
Cơ chế hoạt động bên trong

Đối với phép toán chia hai số nguyên (int, long, short, byte), Java tự động loại bỏ hoàn toàn phần thập phân của kết quả:

int a = 5;
int b = 2;
System.out.println(a / b); // Kết quả hiển thị: 2

Để nhận giá trị thực có phần thập phân đầy đủ, bạn bắt buộc phải ép kiểu ít nhất một số hạng sang kiểu số thực (double hoặc float):

System.out.println((double) a / b); // Kết quả hiển thị: 2.5
Cú pháp và quy tắc
result = operand1 operator operand2;

Ví dụ cú pháp:

int sum = a + b;
int product = a * b;
Ví dụ mã nguồn đơn giản
public class Main {
    public static void main(String[] args) {
        int a = 10;
        int b = 3;

        System.out.println("a + b = " + (a + b));
        System.out.println("a - b = " + (a - b));
        System.out.println("a * b = " + (a * b));
        System.out.println("a / b = " + (a / b));
        System.out.println("a % b = " + (a % b));
    }
}

Kết quả đầu ra:

a + b = 13
a - b = 7
a * b = 30
a / b = 3
a % b = 1
Ví dụ thực tế trong dự án

Tính tổng giá trị đơn hàng thực tế của khách hàng:

double price = 120000.0;
int quantity = 3;
double total = price * quantity;
System.out.println("Tổng tiền: " + total);

3.2 Relational Operators (Toán tử so sánh)

Khái niệm và mục đích sử dụng

Toán tử so sánh được sử dụng để đối chiếu, so sánh quan hệ giữa hai giá trị số hạng và trả về kết quả kiểu boolean.

Toán tử Ý nghĩa
==Bằng nhau
!=Khác nhau (Không bằng)
>Lớn hơn
<Nhỏ hơn
>=Lớn hơn hoặc bằng
<=Nhỏ hơn hoặc bằng
Cơ chế hoạt động bên trong

Các biểu thức so sánh này khi được JVM đánh giá sẽ luôn luôn chỉ trả về một trong hai kết quả logic duy nhất: true hoặc false.

int age = 20;
System.out.println(age >= 18); // JVM xuất ra: true
Cú pháp và quy tắc
expression1 relationalOperator expression2

Ví dụ:

age >= 18;
salary > 10000000;
Ví dụ mã nguồn đơn giản
public class Main {
    public static void main(String[] args) {
        int a = 10;
        int b = 20;

        System.out.println(a == b); // false
        System.out.println(a != b); // true
        System.out.println(a < b);  // true
        System.out.println(a >= b); // false
    }
}
Ví dụ thực tế trong dự án

Kiểm tra độ tuổi đăng ký tài khoản hợp lệ của thành viên:

int age = 17;
boolean canRegister = age >= 18;
System.out.println(canRegister); // false

3.3 Logical Operators (Toán tử logic)

Khái niệm và mục đích sử dụng

Toán tử logic dùng để kết hợp nhiều biểu thức so sánh đơn lẻ lại với nhau để tạo ra các điều kiện logic phức tạp.

Toán tử Ý nghĩa logic
&&Toán tử VÀ (AND)
||Toán tử HOẶC (OR)
!Toán tử PHỦ ĐỊNH (NOT)
Cơ chế hoạt động bên trong

1. Bảng chân trị toán tử AND (&&)

Chỉ trả về true khi tất cả mọi điều kiện thành phần đều đúng.

ABA && B
truetruetrue
truefalsefalse
falsetruefalse
falsefalsefalse

2. Bảng chân trị toán tử OR (||)

Trả về true khi chỉ cần ít nhất một điều kiện thành phần đúng.

ABA || B
truetruetrue
truefalsetrue
falsetruetrue
falsefalsefalse

3. Cơ chế đánh giá ngắn mạch (Short-Circuit Evaluation)

Đây là cơ chế tối ưu hiệu năng chạy của JVM. Ví dụ:

int a = 5;
if (a > 10 && a / 0 > 1) {
    System.out.println("Hello");
}

Ở điều kiện trên, vế trái a > 10 trả về false. Vì đây là toán tử &&, JVM biết chắc kết quả cuối cùng luôn là false nên lập tức dừng lại, bỏ qua không kiểm tra vế phải a / 0 > 1. Nhờ vậy, chương trình không bị lỗi chia cho 0 (ArithmeticException) làm sập hệ thống.

Cú pháp và quy tắc
condition1 && condition2
condition1 || condition2
!condition
Ví dụ mã nguồn đơn giản
public class Main {
    public static void main(String[] args) {
        int age = 20;
        boolean verified = true;
        boolean canLogin = age >= 18 && verified;
        System.out.println(canLogin); // true
    }
}
Ví dụ thực tế trong dự án

Kiểm tra quyền truy cập vào chức năng admin quản lý:

boolean isAdmin = true;
boolean isActive = true;

if (isAdmin && isActive) {
    System.out.println("Hệ thống cho phép truy cập admin");
}

3.4 Assignment Operators (Toán tử gán)

Khái niệm và mục đích sử dụng

Toán tử gán dùng để lưu trữ giá trị từ vế phải vào ô nhớ của biến ở vế trái. Hỗ trợ các cú pháp gán tích hợp số học giúp mã nguồn ngắn gọn hơn.

Toán tử Cú pháp viết tương đương
=a = b (Gán trực tiếp)
+=a = a + b
-=a = a - b
*=a = a * b
/=a = a / b
%=a = a % b
Cơ chế hoạt động bên trong

Khi compiler gặp câu lệnh gán phức a += 5;, nó sẽ biên dịch ngầm thành a = a + 5;, lấy giá trị hiện tại của biến, cộng thêm 5 và cập nhật ghi đè lại vào ô nhớ của biến đó.

Cú pháp và quy tắc
variable operator value;

Ví dụ:

score += 10;
quantity -= 2;
Ví dụ mã nguồn đơn giản
public class Main {
    public static void main(String[] args) {
        int score = 50;
        score += 10; // score = 60
        score -= 5;  // score = 55
        score *= 2;  // score = 110
        System.out.println(score); // 110
    }
}
Ví dụ thực tế trong dự án

Cập nhật lại số lượng sản phẩm tăng thêm trong giỏ hàng:

int quantity = 1;
quantity += 2; // Khách hàng tăng thêm 2 sản phẩm
System.out.println("Số lượng sản phẩm: " + quantity);

3.5 Ternary Operator (Toán tử ba ngôi)

Khái niệm và mục đích sử dụng

Toán tử ba ngôi là cú pháp rút gọn viết trên một dòng của câu lệnh điều kiện if-else cơ bản nhằm trả về một giá trị duy nhất.

Cơ chế hoạt động bên trong

Dòng code:

String result = age >= 18 ? "Adult" : "Child";

Thực tế khi biên dịch sang mã máy hoạt động hoàn toàn tương đồng với khối điều kiện:

String result;
if (age >= 18) {
    result = "Adult";
} else {
    result = "Child";
}
Cú pháp và quy tắc
variable = condition ? value1 : value2;

Quy tắc lập trình sạch (Clean Code): Chỉ nên sử dụng toán tử 3 ngôi khi biểu thức điều kiện ngắn gọn, rõ ràng. Tuyệt đối không lồng nhiều toán tử 3 ngôi vào nhau (ví dụ: a ? b : c ? d : e) vì sẽ làm code cực kỳ rối và khó bảo trì.

Ví dụ mã nguồn đơn giản
public class Main {
    public static void main(String[] args) {
        int age = 20;
        String status = age >= 18 ? "Đủ tuổi" : "Chưa đủ tuổi";
        System.out.println(status); // Đủ tuổi
    }
}
Ví dụ thực tế trong dự án

Hiển thị trạng thái của tài khoản người dùng hoặc xử lý tránh lỗi tham chiếu rỗng (NullPointerException):

boolean active = true;
String accountStatus = active ? "Tài khoản đang hoạt động" : "Tài khoản đã bị khóa";

// Tránh lỗi null khi hiển thị ảnh đại diện (avatar)
String avatar = user.getAvatar() != null ? user.getAvatar() : "default-avatar.png";
