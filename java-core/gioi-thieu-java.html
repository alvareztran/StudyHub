1.1 Java là gì

Khái niệm và mục đích sử dụng

Java là một ngôn ngữ lập trình hướng đối tượng (Object-Oriented Programming - OOP) được phát triển bởi Sun Microsystems vào năm 1995 và hiện nay thuộc sở hữu của Oracle.

Java được thiết kế theo triết lý:

"Write Once, Run Anywhere" (WORA)
Viết một lần, chạy ở mọi nơi.

Điều này có nghĩa là chương trình Java sau khi biên dịch có thể chạy trên mọi hệ điều hành như Windows, Linux hoặc macOS mà không cần biên dịch lại.

Java được sử dụng rộng rãi trong:

  • Phát triển ứng dụng doanh nghiệp (Enterprise Application)
  • Backend Web với Spring Boot
  • Ứng dụng Android
  • Hệ thống ngân hàng, tài chính
  • Hệ thống thương mại điện tử
  • Big Data và Distributed System
Cơ chế hoạt động bên trong

Khác với C/C++, Java không biên dịch trực tiếp thành mã máy của hệ điều hành.

Java Source Code (.java) ↓ Compiler (javac) ↓ Bytecode (.class) ↓ JVM (Máy ảo Java) ↓ Machine Code của hệ điều hành (Windows/Linux/Mac)

Nhờ JVM mà cùng một file .class có thể chạy trên nhiều hệ điều hành khác nhau.

Cú pháp và quy tắc

Tên file Java phải trùng với tên của class public. Ví dụ:

public class Main {
    // Nội dung class
}

File chứa đoạn code này bắt buộc phải được lưu với tên trùng khớp hoàn toàn:

Main.java
Ví dụ mã nguồn đơn giản
public class Main {
    public static void main(String[] args) {
        System.out.println("Hello Java");
    }
}

Kết quả đầu ra:

Hello Java
Ví dụ thực tế trong dự án

Trong thực tế phát triển dự án Spring Boot, tệp khởi chạy ứng dụng được cấu hình như sau:

@SpringBootApplication
public class Application {
    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}

Đây chính là điểm bắt đầu (entry point) của toàn bộ ứng dụng doanh nghiệp Spring Boot.

1.2 JDK, JRE, JVM

Khái niệm và mục đích sử dụng

1. JVM (Java Virtual Machine)

JVM là máy ảo Java chịu trách nhiệm thực thi bytecode. Nhiệm vụ chính:

  • Chạy file .class
  • Quản lý và cấp phát bộ nhớ
  • Dọn rác bộ nhớ tự động (Garbage Collection)
  • Chuyển đổi Bytecode thành mã máy gốc cụ thể của hệ điều hành

2. JRE (Java Runtime Environment)

JRE cung cấp môi trường cần thiết để chạy một ứng dụng Java. Bao gồm:

  • JVM (Máy ảo Java)
  • Thư viện Java chuẩn hóa (Core libraries)

Mục đích: Chỉ dùng để chạy chương trình Java có sẵn, không chứa trình biên dịch để lập trình viên phát triển mã nguồn.

3. JDK (Java Development Kit)

JDK là bộ công cụ hoàn chỉnh để lập trình và phát triển phần mềm Java. Bao gồm:

  • JRE
  • Trình biên dịch javac
  • Công cụ đóng gói jar, Debugger, Javadoc...

Mục đích: Viết, biên dịch và chạy chương trình Java trong quá trình phát triển dự án.

Cơ chế hoạt động bên trong

Mối quan hệ bao đóng giữa các thành phần:

JDK (Development Tools: javac, jar...) └── JRE (Core Libraries) └── JVM (Virtual Machine Execution)

Công thức biểu diễn:

JDK = JRE + Development Tools
JRE = JVM + Libraries
Cú pháp và quy tắc

Kiểm tra sự tồn tại và phiên bản của môi trường JRE/JVM cài đặt trên hệ thống:

java -version

Kiểm tra trình biên dịch JDK cài đặt trên hệ thống máy phát triển:

javac -version
Ví dụ mã nguồn đơn giản

Để biên dịch file mã nguồn bằng JDK:

javac Main.java

Sau khi lệnh biên dịch hoàn thành, hệ thống sinh ra file Bytecode:

Main.class

Để thực thi file bytecode vừa tạo bằng JRE:

java Main
Ví dụ thực tế trong dự án

Khi chạy và phát triển dự án Spring Boot thông qua Maven:

mvn spring-boot:run

Hoặc khi đóng gói và triển khai ứng dụng trên máy chủ server thực tế:

java -jar app.jar

Thực tế JVM trên máy chủ Server sẽ nhận lệnh và trực tiếp thực thi file .jar đã đóng gói.

1.3 Bytecode và cơ chế chạy Java

Khái niệm và mục đích sử dụng

Bytecode là mã lệnh trung gian chứa các chỉ thị nhị phân dạng tối giản được tạo ra sau khi lập trình viên thực hiện biên dịch file mã nguồn .java. Bytecode được lưu với phần mở rộng tệp là .class và hoàn toàn không phụ thuộc vào hệ điều hành nền.

Cơ chế hoạt động bên trong

Ví dụ dòng lệnh xuất chuỗi:

System.out.println("Hello");

Quy trình thực thi thông qua sơ đồ:

Main.java (Source Code) ↓ javac (Biên dịch) ↓ Main.class (Bytecode trung gian độc lập hệ điều hành) ↓ JVM (Đọc bytecode, dịch sang tập lệnh tương ứng của OS) ↓ CPU thực thi lệnh máy vật lý
Cú pháp và quy tắc

Thực hiện biên dịch tạo bytecode:

javac Main.java

Sử dụng công cụ tháo rời bytecode (disassembler) tích hợp sẵn trong JDK để đọc các lệnh bytecode thực tế:

javap -c Main
Ví dụ mã nguồn đơn giản
public class Main {
    public static void main(String[] args) {
        int a = 5;
        int b = 10;
        System.out.println(a + b);
    }
}
Ví dụ thực tế trong dự án

Trong các dự án quản lý bằng công cụ Maven, khi bạn thực hiện câu lệnh đóng gói:

mvn package

Toàn bộ các file Bytecode .class sẽ được tự động biên dịch và gom cụm lưu trữ tại thư mục đích:

target/classes

1.4 Compile và Interpret

Khái niệm và mục đích sử dụng

1. Compile (Biên dịch)

Là quá trình quét và dịch toàn bộ tệp mã nguồn Java (.java) sang định dạng Bytecode trung gian (.class). Công cụ đảm nhận là trình biên dịch javac.

2. Interpret (Thông dịch)

Là quá trình máy ảo JVM đọc và thông dịch từng dòng lệnh Bytecode thành mã máy vật lý tương ứng của hệ điều hành ngay tại thời điểm runtime. Công cụ đảm nhận là bộ **JVM Interpreter**.

Java là ngôn ngữ đặc sắc khi kết hợp đồng thời cả hai cơ chế: **Vừa Compile vừa Interpret** để cân bằng giữa tốc độ khởi động và tính độc lập nền tảng.

Cơ chế hoạt động bên trong
[Mã nguồn .java] │ (Compile bằng javac) ▼ [Bytecode .class] │ ├───> Interpreter (Dịch từng dòng lệnh) ──> Mã máy chạy ngay │ └───> JIT Compiler (Dịch khối mã nóng) ──> Mã máy lưu Cache (Tối ưu hiệu năng)

JIT Compiler (Just-In-Time): Trong lúc chạy, nếu phát hiện ra đoạn code nào được gọi chạy đi chạy lại nhiều lần (Hot Code), JVM sẽ dùng JIT Compiler để biên dịch trực tiếp khối code đó thẳng thành mã máy vật lý và lưu vào bộ nhớ cache, giúp các lần gọi sau chạy với tốc độ tối đa.

Cú pháp và quy tắc

Thực thi biên dịch (Compile):

javac Main.java

Thực thi chạy chương trình (Interpret + JIT):

java Main
Ví dụ mã nguồn đơn giản
public class Main {
    public static void main(String[] args) {
        System.out.println("Compile and Run");
    }
}
Ví dụ thực tế trong dự án

Luồng quy trình triển khai phần mềm chuẩn doanh nghiệp:

Lập trình viên (Viết code) ↓ Build project (Biên dịch tạo File JAR chứa Bytecode) ↓ Deploy lên Server chạy Linux OS ↓ JVM Linux tiến hành thông dịch và tối ưu JIT thực thi ứng dụng

1.5 Cấu trúc chương trình Java đầu tiên

Khái niệm và mục đích sử dụng

Mỗi câu lệnh và từ khóa trong chương trình Java đầu tiên đều có quy định bắt buộc về cú pháp và cấu trúc để JVM nhận diện điểm bắt đầu thực thi.

Cơ chế hoạt động bên trong

Khi bạn gọi lệnh khởi chạy:

java Main

Hệ thống JVM sẽ tiến hành tìm kiếm phương thức bắt buộc có chữ ký (signature) chính xác:

public static void main(String[] args)

Nếu không tìm thấy phương thức này, JVM sẽ báo lỗi khởi chạy và dừng chương trình lập tức.

Cú pháp và quy tắc
public class Main {
    public static void main(String[] args) {
        // Lệnh thực thi viết tại đây
    }
}
  • public: Cho phép JVM truy cập class/phương thức từ phạm vi bên ngoài hệ thống.
  • static: Cho phép JVM gọi chạy phương thức trực tiếp mà không cần khởi tạo đối tượng bằng từ khóa `new`.
  • void: Hàm thực thi tác vụ trực tiếp và không trả về bất kỳ giá trị nào cho JVM.
  • main: Tên phương thức mặc định làm điểm khởi chạy ứng dụng được JVM quy ước sẵn.
  • String[] args: Mảng chứa các chuỗi tham số truyền vào từ dòng lệnh khi chạy chương trình.
Ví dụ mã nguồn đơn giản
public class Main {
    public static void main(String[] args) {
        System.out.println("Xin chào Java");
    }
}
Ví dụ thực tế trong dự án

Truyền và xử lý tham số cấu hình khởi chạy từ bên ngoài dòng lệnh:

java Main Bang 20

Code xử lý đọc tham số:

public class Main {
    public static void main(String[] args) {
        System.out.println("Tên: " + args[0]);
        System.out.println("Tuổi: " + args[1]);
    }
}

Kết quả hiển thị:

Tên: Bang
Tuổi: 20

Trong thực tế, Spring Boot cũng sử dụng cơ chế đọc mảng args này để nhận các cấu hình môi trường bên ngoài khi khởi chạy.
