4.1 if - else

Khái niệm và mục đích sử dụng

Cấu trúc điều kiện if - else cho phép chương trình rẽ sang các nhánh xử lý khác nhau tùy thuộc vào điều kiện biểu thức kiểm tra.

Khi điều kiện trả về giá trị logic là true thì khối lệnh trong khối if được thực thi. Ngược lại, nếu giá trị logic trả về là false, chương trình sẽ bỏ qua và thực hiện khối lệnh định nghĩa trong phần else.

Ứng dụng thực tế phổ biến:

  • Kiểm tra tính hợp lệ của dữ liệu đầu vào (Validation)
  • Phân quyền người dùng (Authorization)
  • Kiểm tra trạng thái nghiệp vụ trước khi cập nhật DB
Cơ chế hoạt động bên trong

JVM tiến hành đánh giá biểu thức bên trong dấu ngoặc đơn của câu lệnh if. Biểu thức này bắt buộc phải trả về kiểu logic boolean.

Biểu thức điều kiện / \ (True) / \ (False) v v Chạy khối if Chạy khối else \ / \---><-----/ ↓ Chạy tiếp code phía sau

Khi gặp chuỗi điều kiện if - else if - else liên tiếp, JVM quét tuần tự từ trên xuống dưới. Ngay khi tìm thấy điều kiện khớp true đầu tiên, nó lập tức nhảy vào xử lý khối lệnh đó và bỏ qua hoàn toàn các nhánh else if hay else còn lại phía dưới.

Cú pháp và quy tắc

1. Cấu trúc if đơn:

if (condition) {
    // Thực thi khi condition là true
}

2. Cấu trúc if - else đôi:

if (condition) {
    // Thực thi khi condition là true
} else {
    // Thực thi khi condition là false
}

3. Cấu trúc lồng ghép nhiều nhánh if - else if - else:

if (condition1) {
    // Khối 1
} else if (condition2) {
    // Khối 2
} else {
    // Khối mặc định khi không có điều kiện nào đúng
}

Quy tắc biên dịch nghiêm ngặt: Biểu thức điều kiện trong if(...) bắt buộc phải có kiểu trả về là boolean. Java không tự động coi các số nguyên khác không hoặc chuỗi không rỗng là true như ngôn ngữ JavaScript/C++.

int age = 18;
// LỖI BIÊN DỊCH: if (age) { ... }
if (age >= 18) { ... } // Hợp lệ
Ví dụ mã nguồn đơn giản
public class Main {
    public static void main(String[] args) {
        int age = 20;

        if (age >= 18) {
            System.out.println("Đủ tuổi");
        } else {
            System.out.println("Chưa đủ tuổi");
        }
    }
}
Ví dụ thực tế trong dự án

Xử lý ném ngoại lệ khi dữ liệu rỗng hoặc tài khoản bị khóa trong nghiệp vụ API:

if (user == null) {
    throw new RuntimeException("Không tìm thấy thông tin tài khoản");
}

if (!user.isActive()) {
    throw new RuntimeException("Tài khoản hiện đang bị khóa tạm thời");
}

// Kiểm tra so sánh mật khẩu mã hóa trong Spring Security:
if (passwordEncoder.matches(password, user.getPassword())) {
    // Tiến hành cấp phát token đăng nhập thành công
}

4.2 switch

Khái niệm và mục đích sử dụng

Cấu trúc rẽ nhánh switch được tối ưu hóa cho các tác vụ so sánh một biến đơn lẻ với nhiều giá trị không đổi khác nhau (cố định), giúp mã nguồn sạch hơn so với viết quá nhiều câu lệnh if-else if chồng chéo.

Cơ chế hoạt động bên trong

JVM lấy giá trị của biểu thức truyền vào, so khớp trực tiếp với giá trị khai báo ở các nhánh case. Khi tìm thấy case khớp, nó thực hiện các dòng lệnh bên trong.

Hiện tượng Fall-Through: Nếu ở cuối khối xử lý của case không có từ khóa ngắt lệnh break, JVM sẽ tiếp tục "trôi" tự do chạy tất cả các case bên dưới bất chấp giá trị so sánh có khớp hay không. Đây là lỗi logic thường gặp của lập trình viên.

Cú pháp và quy tắc

1. Cấu trúc Switch truyền thống:

switch (expression) {
    case value1:
        // Lệnh xử lý 1
        break; // Thoát switch
    case value2:
        // Lệnh xử lý 2
        break;
    default:
        // Xử lý mặc định khi không khớp case nào
}

2. Cấu trúc Switch Expression hiện đại (Hỗ trợ từ Java 14+ giúp chống lỗi quên viết break):

switch (day) {
    case "MONDAY" -> System.out.println("Thứ Hai");
    case "TUESDAY" -> System.out.println("Thứ Ba");
    default -> System.out.println("Ngày chưa xác định");
}
Ví dụ mã nguồn đơn giản
public class Main {
    public static void main(String[] args) {
        int month = 2;

        switch (month) {
            case 1:
                System.out.println("January");
                break;
            case 2:
                System.out.println("February");
                break;
            case 3:
                System.out.println("March");
                break;
            default:
                System.out.println("Unknown");
        }
    }
}
Ví dụ thực tế trong dự án

Kiểm tra quyền truy cập menu dashboard theo phân quyền người dùng (Role):

switch (user.getRole()) {
    case ADMIN:
        showAdminDashboard();
        break;
    case STAFF:
        showStaffDashboard();
        break;
    case CUSTOMER:
        showCustomerDashboard();
        break;
}

4.3 for

Khái niệm và mục đích sử dụng

Vòng lặp for được sử dụng để thực hiện lặp đi lặp lại một khối lệnh với số lần lặp xác định trước.

Cơ chế hoạt động bên trong

Quy trình chạy tuần tự của vòng lặp for trong JVM:

Khởi tạo giá trị biến đếm (int i = 0) -> Chạy duy nhất 1 lần đầu ↓ ┌───> Kiểm tra điều kiện (i < N) │ ↓ (Nếu True) │ Thực thi khối lệnh │ ↓ │ Cập nhật biến đếm (i++) └────────────────────┘ (Quay lại kiểm tra)

Khi điều kiện kiểm tra trả về false, vòng lặp dừng ngay lập tức.

Cú pháp và quy tắc
for (initialization; condition; update) {
    // Khối code lặp
}
Ví dụ mã nguồn đơn giản
public class Main {
    public static void main(String[] args) {
        for (int i = 1; i <= 5; i++) {
            System.out.println(i);
        }
    }
}

Kết quả đầu ra:

1
2
3
4
5
Ví dụ thực tế trong dự án

Duyệt qua danh sách đối tượng đọc từ Database và in ra thông tin email của họ:

List<User> users = userRepository.findAll();

// Duyệt bằng vòng lặp For-Each cải tiến (Enhanced For Loop)
for (User user : users) {
    System.out.println(user.getEmail());
}

4.4 while

Khái niệm và mục đích sử dụng

Vòng lặp while dùng để lặp lại một khối lệnh khi không biết trước chính xác số lần lặp. Vòng lặp sẽ tiếp tục chạy miễn là biểu thức điều kiện đầu vào còn đúng (true).

Cơ chế hoạt động bên trong

JVM tiến hành kiểm tra điều kiện trước. Nếu điều kiện đúng, khối lệnh trong thân while mới được chạy. Nếu điều kiện sai ngay từ đầu, khối lệnh sẽ không được chạy bất kỳ lần nào.

Cú pháp và quy tắc
while (condition) {
    // Khối code lặp
    // Bắt buộc phải có dòng code thay đổi trạng thái điều kiện để tránh lặp vô tận
}
Ví dụ mã nguồn đơn giản
public class Main {
    public static void main(String[] args) {
        int i = 1;
        while (i <= 5) {
            System.out.println(i);
            i++; // Tăng điều kiện
        }
    }
}
Ví dụ thực tế trong dự án

Đọc tuần tự từng dòng văn bản từ file (không biết trước file dài bao nhiêu dòng):

String line;
while ((line = reader.readLine()) != null) {
    System.out.println(line); // Đọc cho đến khi hết file (null)
}

// Đọc kết nối Socket nhận dữ liệu liên tục từ Client gửi về:
while (socket.isConnected()) {
    byte[] data = receiveBuffer();
    process(data);
}

4.5 do while

Khái niệm và mục đích sử dụng

Vòng lặp do while tương tự như while, tuy nhiên điểm khác biệt lớn nhất là nó đảm bảo khối lệnh được **thực thi ít nhất một lần** trước khi kiểm tra điều kiện lặp.

Cơ chế hoạt động bên trong

JVM chạy trực tiếp toàn bộ khối code nằm trong nhãn do { ... } trước. Sau đó, nó mới đi xuống nhãn while(condition) để đánh giá điều kiện. Nếu đúng, nó tiếp tục quay lên lặp; nếu sai, nó thoát vòng lặp.

  • Vòng lặp while: Kiểm tra điều kiện rồi mới chạy (Check-first).
  • Vòng lặp do-while: Chạy trước rồi mới kiểm tra điều kiện (Execute-first).
Cú pháp và quy tắc
do {
    // Khối code lặp
} while (condition); // Bắt buộc phải có dấu chấm phẩy ";" ở cuối cùng
Ví dụ mã nguồn đơn giản
public class Main {
    public static void main(String[] args) {
        int number = 1;
        do {
            System.out.println(number);
            number++;
        } while (number <= 5);
    }
}
Ví dụ thực tế trong dự án

Thiết lập màn hình hiển thị menu console chọn lựa dịch vụ và kiểm tra phản hồi người dùng nhập vào:

int choice;
do {
    System.out.println("--- MENU LỰA CHỌN ---");
    System.out.println("1. Đăng nhập hệ thống");
    System.out.println("2. Đăng ký tài khoản");
    System.out.println("0. Thoát chương trình");
    
    choice = scanner.nextInt(); // Nhập lựa chọn
} while (choice != 0); // Vòng lặp chỉ dừng khi người dùng chủ động chọn 0

4.6 break và continue

Khái niệm và mục đích sử dụng
  • break: Dùng để thoát hoàn toàn khỏi cấu trúc vòng lặp hoặc cấu trúc switch hiện tại chứa nó lập tức.
  • continue: Bỏ qua phần mã nguồn xử lý phía dưới của lượt lặp hiện tại để chuyển sang lượt lặp kế tiếp lập tức.
Cơ chế hoạt động bên trong

Khi gặp break, JVM lập tức hủy bỏ vòng lặp và nhảy dòng lệnh thực thi xuống dòng tiếp theo nằm ngoài vòng lặp.

Khi gặp continue, JVM bỏ qua phần lệnh còn lại, quay ngược lại đầu vòng lặp để cập nhật biến đếm (đối với for) hoặc kiểm tra điều kiện lặp (đối với while).

Cú pháp và quy tắc
break;
continue;
Ví dụ mã nguồn đơn giản

1. Ví dụ dùng break (dừng khi gặp số 5):

for (int i = 1; i <= 10; i++) {
    if (i == 5) {
        break; // Thoát vòng lặp
    }
    System.out.println(i);
} // Chỉ in ra: 1, 2, 3, 4

2. Ví dụ dùng continue (bỏ qua số 3):

for (int i = 1; i <= 5; i++) {
    if (i == 3) {
        continue; // Bỏ qua in ra số 3, tiếp tục vòng lặp
    }
    System.out.println(i);
} // In ra: 1, 2, 4, 5
Ví dụ thực tế trong dự án

1. Tối ưu hóa tìm kiếm phần tử trong danh sách:

User foundUser = null;
for (User user : users) {
    if (user.getId().equals(targetId)) {
        foundUser = user;
        break; // Tìm thấy thì dừng ngay lập tức để tiết kiệm tài nguyên CPU
    }
}

2. Lọc bỏ và chỉ xử lý các tài khoản người dùng đang hoạt động:

for (User user : users) {
    if (!user.isActive()) {
        continue; // Tài khoản bị khóa thì bỏ qua, chuyển sang tài khoản kế tiếp
    }
    sendEmail(user); // Chỉ gửi email cho người dùng đang active
}
