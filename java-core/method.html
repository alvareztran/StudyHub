5.1 Khai báo Method

Khái niệm và mục đích sử dụng

Method (Phương thức / Hàm) là một tập hợp các khối câu lệnh được viết sẵn để thực thi một công việc nghiệp vụ cụ thể.

Mục đích sử dụng:

  • Tránh viết trùng lặp mã nguồn ở nhiều nơi (Don't Repeat Yourself - DRY)
  • Chia nhỏ một bài toán lớn phức tạp thành các phương thức nhỏ để dễ kiểm thử, gỡ lỗi (Clean Code)
  • Nâng cao tính tái sử dụng và quản lý mã nguồn khoa học
Cơ chế hoạt động bên trong

Khi gọi một phương thức, JVM sẽ tự động cấp phát một phân vùng nhớ riêng gọi là **Stack Frame** đặt lên trên đỉnh vùng nhớ **Call Stack**.

[Call Stack] ┌─────────────────────────────┐ │ Stack Frame của calculate() │ (Đang thực thi calculate) ├─────────────────────────────┤ │ Stack Frame của main() │ (Tạm ngưng đợi calculate xong) └─────────────────────────────┘

Stack Frame này chứa toàn bộ các tham số truyền vào và các biến cục bộ khai báo bên trong phương thức. Ngay khi phương thức chạy xong tác vụ (gặp lệnh return hoặc hết hàm), JVM sẽ tự động xóa (pop) Stack Frame này ra khỏi Call Stack để trả lại không gian bộ nhớ.

Cú pháp và quy tắc
access_modifier return_type method_name(parameters) {
    // Khối code xử lý
}

Mô tả chi tiết các thành phần:

Thành phần Mô tả chi tiết
access_modifierPhạm vi truy cập (như public, private, protected, default)
return_typeKiểu dữ liệu trả về sau khi chạy xong (int, double, String...). Dùng void nếu không trả về gì.
method_nameTên phương thức (bắt buộc viết theo chuẩn camelCase)
parametersDanh sách các tham số đầu vào (cách nhau bởi dấu phẩy)
Ví dụ mã nguồn đơn giản
public class Main {
    public static void sayHello() {
        System.out.println("Hello World");
    }

    public static void main(String[] args) {
        sayHello(); // Gọi phương thức tĩnh
    }
}
Ví dụ thực tế trong dự án

Tìm kiếm thông tin người dùng theo ID trong Spring Boot Service:

public User findUserById(Long id) {
    return userRepository.findById(id)
            .orElseThrow(() -> new RuntimeException("Không tìm thấy người dùng này"));
}

5.2 Tham số và giá trị trả về

Khái niệm và mục đích sử dụng
  • Tham số (Parameter): Là các biến khai báo ở signature của hàm để nhận dữ liệu đầu vào.
  • Giá trị trả về (Return Value): Là kết quả tính toán đầu ra mà hàm trả lại cho nơi gọi hàm sau khi xử lý xong nhiệm vụ.
Cơ chế hoạt động bên trong

Khi ta gọi lệnh: int total = add(5, 10);

JVM sao chép hai giá trị 5 và 10 vào các biến cục bộ tương ứng của Stack Frame phương thức add(). Sau khi cộng, lệnh return đẩy kết quả về vế trái và hủy bỏ Stack Frame của hàm.

Cú pháp và quy tắc

Cài đặt tham số đầu vào và nạp giá trị đầu ra:

public static int sum(int a, int b) {
    return a + b; // Phải khớp với kiểu trả về "int"
}
Ví dụ mã nguồn đơn giản
public class Main {
    public static int add(int a, int b) {
        return a + b;
    }

    public static void main(String[] args) {
        int result = add(10, 20);
        System.out.println(result); // 30
    }
}
Ví dụ thực tế trong dự án

Kiểm tra email trùng lặp khi đăng ký tài khoản:

public boolean existsByEmail(String email) {
    return userRepository.existsByEmail(email); // Trả về boolean true/false
}

// Sử dụng check logic:
if (existsByEmail(email)) {
    throw new RuntimeException("Email đã tồn tại trên hệ thống!");
}

5.3 Method Overloading (Nạp chồng phương thức)

Khái niệm và mục đích sử dụng

Method Overloading là khả năng một Class khai báo nhiều phương thức có cùng tên gọi hoàn toàn nhưng bắt buộc phải khác nhau về danh sách tham số (khác số lượng tham số, khác kiểu dữ liệu, hoặc khác thứ tự kiểu dữ liệu).

Mục đích: Giúp lập trình viên có trải nghiệm gọi hàm tự nhiên với các kiểu dữ liệu đầu vào khác nhau mà không cần đặt tên hàm khác biệt rối rắm.

Cơ chế hoạt động bên trong

Trình biên dịch Java (compiler) tự động đối chiếu kiểu dữ liệu và số lượng tham số truyền vào tại thời điểm viết code để gắn kết phương thức phù hợp chính xác. Đây được gọi là Đa hình thời kỳ biên dịch (Compile Time Polymorphism) hoặc Static Binding.

Cú pháp và quy tắc

Hợp lệ (Overloading đúng):

int add(int a, int b);
double add(double a, double b);
int add(int a, int b, int c);

Không hợp lệ (Overloading sai):

int add(int a, int b);
double add(int a, int b); // LỖI: Java không cho phép overload chỉ khác kiểu dữ liệu trả về
Ví dụ mã nguồn đơn giản
public class Calculator {
    public int add(int a, int b) {
        return a + b;
    }
    public double add(double a, double b) {
        return a + b;
    }
    public int add(int a, int b, int c) {
        return a + b + c;
    }
}
Ví dụ thực tế trong dự án

Phương thức String.valueOf() tích hợp sẵn trong bộ SDK của Java chính là ví dụ kinh điển của Overloading:

String.valueOf(10);     // Nhận int
String.valueOf(10.5);   // Nhận double
String.valueOf(true);   // Nhận boolean

5.4 Pass By Value (Cơ chế truyền tham chiếu/tham trị)

Khái niệm và mục đích sử dụng

⚠️ Đặc điểm tối thượng: Java chỉ hỗ trợ duy nhất một cơ chế truyền tham số khi gọi hàm là Pass By Value (Truyền giá trị). Java hoàn toàn không hỗ trợ Pass By Reference (truyền tham chiếu thực tế) như ngôn ngữ C++.

Cơ chế hoạt động bên trong

1. Đối với kiểu nguyên thủy (Primitive Types)

JVM sao chép giá trị thực tế của biến truyền vào. Phương thức nhận được một bản sao độc lập. Mọi thay đổi trên biến bản sao trong hàm không tác động gì đến biến gốc ở ngoài hàm.

[Stack Frame main] -> a = 10 │ (Sao chép giá trị thô) ▼ [Stack Frame change] -> x = 10 --(Gán x = 100)--> x = 100 (a gốc vẫn là 10)

2. Đối với kiểu đối tượng (Reference Types)

JVM sao chép **giá trị của địa chỉ tham chiếu** (địa chỉ Heap). Cả biến gốc và biến tham số trong hàm đều cùng trỏ về một ô nhớ thực thể đối tượng trên Heap. Do đó, việc thay đổi thuộc tính của đối tượng thông qua biến trong hàm sẽ thay đổi trực tiếp đối tượng thực tế trên Heap.

[Stack Frame main] -> s (0x88) ───┐ ├──> [HEAP Address 0x88: Student Object] [Stack Frame change] -> student (0x88) ┘ (Thay đổi thuộc tính bên trong sẽ đổi đối tượng chung)
Ví dụ mã nguồn đơn giản

1. Ví dụ với primitive:

public class Main {
    public static void change(int x) {
        x = 100;
    }
    public static void main(String[] args) {
        int a = 10;
        change(a);
        System.out.println(a); // In ra: 10 (Không đổi)
    }
}

2. Ví dụ với Object:

class Student {
    String name;
}

public class Main {
    public static void change(Student student) {
        student.name = "Bang";
    }
    public static void main(String[] args) {
        Student s = new Student();
        s.name = "An";
        change(s);
        System.out.println(s.name); // In ra: "Bang" (Thay đổi thực tế trên Heap)
    }
}

5.5 Recursion (Đệ quy)

Khái niệm và mục đích sử dụng

Đệ quy là kỹ thuật viết code trong đó một phương thức tự gọi lại chính nó. Thích hợp giải quyết các bài toán có thể phân tách thành các bài toán con nhỏ hơn có cấu trúc đồng dạng.

Cơ chế hoạt động bên trong

Mỗi lần gọi đệ quy, JVM lại tạo ra một Stack Frame mới xếp chồng lên Call Stack. Nếu đệ quy gọi quá nhiều lần mà không chạm được điều kiện dừng (Base Case), bộ nhớ Stack sẽ bị quá tải hoàn toàn dẫn đến lỗi nghiêm trọng:

java.lang.StackOverflowError
Cú pháp và quy tắc

Mọi hàm đệ quy bắt buộc phải đáp ứng đủ 2 phần:

  1. Base Case (Điểm dừng): Điều kiện để dừng đệ quy không cho gọi tiếp.
  2. Recursive Case (Bước đệ quy): Gọi lại hàm với tham số nhỏ hơn.
Ví dụ mã nguồn đơn giản

Tính giai thừa $n!$ bằng đệ quy:

public class Main {
    public static int factorial(int n) {
        if (n == 1) { // Điều kiện dừng
            return 1;
        }
        return n * factorial(n - 1); // Bước đệ quy
    }

    public static void main(String[] args) {
        System.out.println(factorial(5)); // 120
    }
}
Ví dụ thực tế trong dự án

Duyệt in toàn bộ danh sách tệp tin nằm sâu trong cấu trúc thư mục nhiều cấp:

public void scanFolder(File folder) {
    for (File file : folder.listFiles()) {
        if (file.isDirectory()) {
            scanFolder(file); // Tự gọi lại đệ quy để đào sâu vào thư mục con
        }
        System.out.println(file.getName());
    }
}

5.6 Varargs (Tham số có độ dài biến đổi)

Khái niệm và mục đích sử dụng

Varargs cho phép thiết lập phương thức nhận vào số lượng đối số tùy ý tại nơi gọi hàm mà không cần giới hạn trước số lượng tham số cố định.

Cơ chế hoạt động bên trong

Tại thời điểm biên dịch, trình biên dịch Java sẽ tự động gom toàn bộ các đối số lẻ tẻ truyền vào thành một mảng (Array) dạng thô để truyền vào phương thức. Bản chất Varargs bên trong phương thức chính là một mảng.

Cú pháp và quy tắc
datatype... variableName

Quy tắc bắt buộc:

  • Chỉ được chứa duy nhất 1 tham số Varargs trong signature của phương thức.
  • Tham số Varargs bắt buộc phải đứng ở vị trí cuối cùng trong danh sách tham số.
// ĐÚNG:
public void test(String label, int... numbers)
// SAI LỚN:
public void test(int... numbers, String label)
Ví dụ mã nguồn đơn giản
public class Main {
    public static int sum(int... numbers) {
        int total = 0;
        for (int num : numbers) {
            total += num;
        }
        return total;
    }

    public static void main(String[] args) {
        System.out.println(sum(1, 2));        // 3
        System.out.println(sum(1, 2, 3, 4));  // 10
    }
}
Ví dụ thực tế trong dự án

Phương thức in định dạng System.out.printf() sử dụng Varargs nhận vào các đối số động đi kèm:

System.out.printf("Tên: %s, Tuổi: %d", "Bằng", 21);

// Chữ ký hàm thực tế trong SDK Java:
// printf(String format, Object... args)
