13.1 Lớp File

Khái niệm và mục đích sử dụng

Lớp File nằm trong gói thư viện java.io đại diện cho siêu dữ liệu (metadata) mô tả đường dẫn tệp tin hoặc thư mục vật lý trong hệ điều hành.

⚠️ Đặc điểm quan trọng: Bản thân lớp File **không dùng để thực hiện đọc hay ghi dữ liệu** vào trong tệp tin. Nó chỉ đóng vai trò kiểm tra sự tồn tại, tạo, xóa, kiểm tra quyền và lấy đường dẫn của file hoặc thư mục trên ổ đĩa.

Cơ chế hoạt động bên trong

Khi viết dòng code: File file = new File("data.txt");

JVM chỉ tạo ra duy nhất một đối tượng Java tham chiếu nằm trên Heap mô tả đường dẫn. Nó hoàn toàn **không tạo ra bất kỳ tệp tin nào** trên ổ đĩa cứng thực tế. Tệp tin thật sự chỉ được ghi tạo thành công xuống phần cứng khi bạn thực hiện gọi phương thức: file.createNewFile();

Cú pháp và quy tắc
import java.io.File;
File file = new File("data.txt");

Bảng các phương thức tiện ích phổ biến của File:

Phương thức Mô tả hành vi
exists()Kiểm tra xem file hoặc thư mục có tồn tại thực tế trên đĩa không
createNewFile()Tạo một file trống mới hoàn toàn. Trả về true nếu tạo thành công
delete()Xóa file hoặc thư mục vật lý
mkdir()Tạo thư mục đơn cấp
mkdirs()Tạo thư mục đa cấp (tự tạo cả thư mục cha nếu chưa có)
isFile()Kiểm tra xem đường dẫn có phải là tệp tin không
isDirectory()Kiểm tra xem đường dẫn có phải là thư mục không
listFiles()Lấy danh sách các tệp tin con bên trong thư mục dạng mảng File[]
Ví dụ mã nguồn đơn giản
import java.io.File;
import java.io.IOException;

public class Main {
    public static void main(String[] args) throws IOException {
        File file = new File("hello.txt");
        if (!file.exists()) {
            file.createNewFile(); // Tạo file thực tế trên đĩa cứng
        }
        System.out.println("Tồn tại: " + file.exists()); // true
    }
}
Ví dụ thực tế trong dự án

Tự động khởi tạo thư mục lưu trữ ảnh tải lên (Uploads Folder) nếu chưa tồn tại trên Server:

File uploadFolder = new File("uploads/images");
if (!uploadFolder.exists()) {
    uploadFolder.mkdirs(); // Tạo toàn bộ các cấp thư mục cha-con
}

13.2 InputStream

Khái niệm và mục đích sử dụng

InputStream là lớp cha trừu tượng (abstract class) của mọi luồng dữ liệu đọc thông tin dưới **dạng nhị phân thô (Byte Stream)**.

Sử dụng khi thao tác với các tệp tin phi văn bản (Binary Files) như: Hình ảnh, Video, Nhạc, PDF, tệp tin nén zip, tar hoặc truyền nhận mảng byte qua giao thức mạng Socket.

Cơ chế hoạt động bên trong

Dữ liệu được vận chuyển tuần tự theo dạng luồng ống nước dẫn từng byte từ nguồn file đi vào trong bộ đệm RAM của chương trình:

Tệp tin nhị phân (Đĩa) ───> [ InputStream ] ───> Chương trình (Bộ nhớ RAM)
Cú pháp và ví dụ mã nguồn
import java.io.FileInputStream;
import java.io.IOException;

public class Main {
    public static void main(String[] args) throws IOException {
        FileInputStream fis = new FileInputStream("hello.txt");
        int data;
        // Đọc tuần tự từng byte dữ liệu cho tới khi hết file (trả về -1)
        while ((data = fis.read()) != -1) {
            System.out.print((char) data);
        }
        fis.close(); // Bắt buộc phải đóng luồng sau khi dùng
    }
}
Ví dụ thực tế trong dự án

Tiếp nhận luồng ảnh tải lên từ Client để đẩy lưu trữ lên Cloudinary Cloud:

InputStream inputStream = multipartFile.getInputStream();
// Thực thi copy luồng ghi lưu trữ:
Files.copy(inputStream, destinationPath, StandardCopyOption.REPLACE_EXISTING);

13.3 OutputStream

Khái niệm và mục đích sử dụng

OutputStream là lớp cha trừu tượng của mọi luồng xuất dữ liệu ghi ra bên ngoài dưới **dạng nhị phân thô (Byte Stream)**.

Sử dụng khi cần tạo mới, xuất ghi đè các tệp tin nhị phân như download file từ server, lưu ảnh, kết xuất workbook Excel.

Cơ chế hoạt động bên trong
Chương trình (Bộ nhớ RAM) ───> [ OutputStream ] ───> Tệp tin đích (Đĩa cứng)
Ví dụ mã nguồn đơn giản
import java.io.FileOutputStream;
import java.io.IOException;

public class Main {
    public static void main(String[] args) throws IOException {
        FileOutputStream fos = new FileOutputStream("output.bin");
        String message = "Hello Byte Stream";
        fos.write(message.getBytes()); // Chuyển String sang byte[] rồi ghi xuống file
        fos.close();
    }
}

13.4 Reader

Khái niệm và mục đích sử dụng

Reader là lớp cha trừu tượng của tất cả các luồng đọc dữ liệu dưới **dạng văn bản ký tự (Character Stream)**.

Sự khác biệt cốt lõi giữa Byte Stream và Character Stream:

Tiêu chí phân biệt Byte Stream (InputStream / OutputStream) Character Stream (Reader / Writer)
Đơn vị xử lý cơ bản1 Byte (8 bits)1 Ký tự Unicode (16 bits / 2 bytes)
Mục đích sử dụng chínhẢnh, Video, Zip, PDF, exe...Text, TXT, CSV, JSON, XML, HTML...
Xử lý bảng mã (Encoding)Không tự dịch bảng mãTự động ánh xạ chuyển đổi mã hóa UTF-8, UTF-16...
Ví dụ mã nguồn đơn giản
import java.io.BufferedReader;
import java.io.FileReader;
import java.io.IOException;

public class Main {
    public static void main(String[] args) throws IOException {
        // BufferedReader cung cấp hàm readLine() giúp đọc cả dòng văn bản cực kỳ tiện lợi
        BufferedReader reader = new BufferedReader(new FileReader("hello.txt"));
        String line;
        while ((line = reader.readLine()) != null) {
            System.out.println(line); // In từng dòng văn bản
        }
        reader.close();
    }
}

13.5 Writer

Khái niệm và mục đích sử dụng

Writer là lớp cha trừu tượng chịu trách nhiệm xuất, ghi các luồng dữ liệu dạng văn bản ký tự Unicode xuống thiết bị lưu trữ.

Ví dụ mã nguồn đơn giản
import java.io.FileWriter;
import java.io.IOException;

public class Main {
    public static void main(String[] args) throws IOException {
        FileWriter writer = new FileWriter("report.txt");
        writer.write("Monthly Business Report\n");
        writer.write("Doanh thu tháng này đạt chỉ tiêu đề ra.");
        writer.close(); // Đóng luồng để đẩy toàn bộ dữ liệu từ bộ đệm xuống đĩa
    }
}

Buffered Stream (Luồng đệm nâng cao hiệu năng)

Khi sử dụng các luồng thô như FileInputStream hoặc FileReader, mỗi lần bạn đọc 1 byte hoặc 1 ký tự, JVM đều phải thực hiện một lệnh truy xuất đọc trực tiếp từ ổ đĩa vật lý (Hard Drive I/O). Việc này cực kỳ chậm vì tốc độ của đĩa cứng kém rất xa tốc độ của RAM.

Cơ chế hoạt động của Buffered Stream (Bộ đệm):

Đọc thô: Ổ cứng ──[1 byte]──> CPU ──[1 byte]──> RAM (Quét 1000 lần cho 1000 bytes) Đệm RAM: Ổ cứng ──[Đọc 8KB]──> RAM đệm (Buffer) ──> CPU đọc trực tiếp từ RAM đệm (Cực nhanh)

Hệ thống tự động đọc một khối dữ liệu lớn (mặc định 8KB) nạp vào vùng đệm RAM trước, sau đó chương trình chỉ việc lấy từ RAM ra dùng. Giúp giảm thiểu số lần tương tác trực tiếp với ổ cứng, tăng tốc độ xử lý lên hàng trăm lần.

// Cấu hình chuẩn tối ưu hiệu năng:
BufferedReader reader = new BufferedReader(new FileReader("data.txt"));
BufferedInputStream bis = new BufferedInputStream(new FileInputStream("image.jpg"));

try - with - resources (Quản lý tự động đóng luồng)

Trước Java 7, ta phải đóng luồng thủ công ở khối finally rất rườm rà. Nếu quên đóng, ứng dụng sẽ bị rò rỉ bộ nhớ (Resource Leak) làm tê liệt hệ thống.

Từ Java 7+, ta sử dụng cú pháp try-with-resources để JVM tự động đóng các luồng I/O một cách an toàn tuyệt đối khi kết thúc khối lệnh:

try (
    BufferedReader reader = new BufferedReader(new FileReader("hello.txt"));
    BufferedWriter writer = new BufferedWriter(new FileWriter("copy.txt"))
) {
    String line;
    while ((line = reader.readLine()) != null) {
        writer.write(line);
        writer.newLine();
    }
} catch (IOException e) {
    e.printStackTrace();
} // JVM TỰ ĐỘNG GỌI reader.close() và writer.close() NGẦM ĐỊNH TẠI ĐÂY

Bản đồ hướng dẫn chọn luồng File I/O

Nhiệm vụ nghiệp vụ Luồng khuyên dùng
Đọc các file nhị phân (Ảnh tải lên, PDF, file Zip)InputStream / BufferedInputStream
Ghi xuất các file nhị phân (Ảnh download, file nhị phân)OutputStream / BufferedOutputStream
Đọc file cấu hình văn bản, tệp tin CSV, log dạng textReader / BufferedReader
Xuất ghi báo cáo văn bản, ghi cấu trúc file JSON, XMLWriter / BufferedWriter
Quản lý tài nguyên an toàn tránh rò rỉ bộ nhớLuôn dùng cú pháp try-with-resources
