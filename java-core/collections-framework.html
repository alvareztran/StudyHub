11.1 Collection Hierarchy (Sơ đồ phân cấp bộ khung Collection)

Khái niệm và mục đích sử dụng

Collections Framework là tập hợp các Interfaces, Classes triển khai và Algorithms tối ưu được Java cung cấp sẵn để quản lý, lưu trữ, thao tác với tập hợp dữ liệu có kích thước động co giãn linh hoạt.

So sánh cơ bản giữa Mảng thô (Array) và Collection:

Tiêu chí so sánh Mảng thô (Array) Bộ khung Collection
Kích thước chứaCố định (Fixed size) tại thời điểm khởi tạoĐộng (Dynamic size) tự động co giãn ô nhớ
Kiểu dữ liệu lưuChứa được cả kiểu nguyên thủy và đối tượngChỉ lưu trữ đối tượng (Wrapper / Custom Object)
Tính năng đi kèmÍt phương thức bổ trợ thô sơĐầy đủ thư viện thuật toán sắp xếp, lọc, tìm kiếm
Cơ chế hoạt động bên trong (Sơ đồ phân cấp Hierarchy)
Iterable (Gốc duyệt for-each) ↑ Collection ┌────────────────────┼────────────────────┐ v v v List Set Queue (ArrayList) (HashSet) (PriorityQueue) (LinkedList) (LinkedHashSet) (ArrayDeque) (TreeSet) *Chú ý: Cấu trúc Map lưu Key-Value độc lập, không thừa kế Collection Map (HashMap, TreeMap...)
Cú pháp và quy tắc

Nguyên tắc thiết kế tốt (Loose Coupling): Luôn khai báo biến bằng kiểu Interface cha, và chỉ khởi tạo bằng Class con triển khai thực tế:

List<String> list = new ArrayList<>();
Set<Integer> set = new HashSet<>();
Map<Long, User> userMap = new HashMap<>();

11.2 List (Danh sách tuyến tính)

Khái niệm và mục đích sử dụng

List là một Collection cho phép lưu giữ các phần tử theo đúng **thứ tự thêm vào**, cho phép chứa các phần tử có **giá trị trùng lặp**, và hỗ trợ truy xuất ngẫu nhiên qua chỉ số index.

Cơ chế hoạt động bên trong

1. ArrayList (Mảng động resizable array)

ArrayList lưu dữ liệu bên trong một mảng tĩnh thô. Khi số lượng phần tử vượt quá dung lượng hiện tại, JVM tự động thực thi quy trình giãn nở:

Mảng cũ đầy: [10][20][30][40] (Dung lượng: 4) ↓ JVM tạo mảng mới có kích thước = cũ * 1.5 (Dung lượng mới: 6) ↓ Copy toàn bộ dữ liệu từ mảng cũ sang mảng mới: [10][20][30][40][ ][ ] ↓ Hủy mảng cũ, cập nhật con trỏ trỏ sang mảng mới.

2. LinkedList (Danh sách liên kết đôi - Doubly Linked List)

Mỗi phần tử được bọc trong một Node chứa giá trị và 2 con trỏ liên kết trỏ tới Node liền trước (prev) và Node liền sau (next):

[Node 1: 10] <---> [Node 2: 20] <---> [Node 3: 30]

Không dùng mảng liên tiếp. Ô nhớ được phân bổ phân tán tự do trên Heap.

Khi nào chọn ArrayList hay LinkedList?
Thao tác ArrayList (Mảng động) LinkedList (Liên kết đôi)
Truy cập qua index get(i)$O(1)$ (Cực nhanh nhờ cộng địa chỉ)$O(N)$ (Phải quét duyệt từ đầu node)
Thêm/Xóa ở đuôi add()$O(1)$ (Mặc định thêm cuối mảng)$O(1)$ (Có sẵn con trỏ tail)
Thêm/Xóa ở giữa danh sách$O(N)$ (Phải dịch chuyển các phần tử sau)$O(1)$ (Chỉ cần thay đổi liên kết con trỏ)
Chi phí bộ nhớ RAMThấpCao (Tốn thêm ô nhớ cho các con trỏ prev/next)

Lời khuyên thực tế: Trong 90% bài toán thực tế đi làm, ta ưu tiên sử dụng **`ArrayList`** do tính năng đọc index chiếm ưu thế tuyệt đối.

11.3 Set (Tập hợp không trùng lặp)

Khái niệm và mục đích sử dụng

Set là một Collection chứa các phần tử **không trùng lặp** nhau. Nếu nạp phần tử đã có sẵn vào Set, JVM tự động bỏ qua.

Các Class triển khai chính:

  • HashSet: Không đảm bảo thứ tự của phần tử (Tốc độ nhanh nhất).
  • LinkedHashSet: Giữ nguyên thứ tự thêm vào phần tử.
  • TreeSet: Tự động sắp xếp các phần tử theo thứ tự tăng dần (Sử dụng cấu trúc Red-Black Tree).
Cơ chế hoạt động bên trong của HashSet

Thực chất, HashSet sử dụng cấu trúc của một HashMap chạy ngầm bên dưới. Các giá trị nạp vào Set đóng vai trò là các Key của HashMap, còn Value nhận giá trị là một hằng số dummy giả.

JVM kiểm tra trùng lặp phần tử bằng cách gọi song song hai hàm:

hashCode() và equals()
Ví dụ mã nguồn đơn giản
import java.util.HashSet;
import java.util.Set;

public class Main {
    public static void main(String[] args) {
        Set<String> skills = new HashSet<>();
        skills.add("Java");
        skills.add("Spring");
        skills.add("Java"); // Phần tử trùng lặp bị bỏ qua

        System.out.println(skills); // Xuất ra: [Java, Spring]
    }
}

11.4 Queue (Hàng đợi)

Khái niệm và mục đích sử dụng

Queue là cấu trúc dữ liệu hoạt động chặt chẽ theo nguyên tắc xếp hàng FIFO (First In First Out) - Phần tử nào được nạp vào trước sẽ được trích xuất lấy ra xử lý trước.

Cơ chế hoạt động và cú pháp
offer() (Vào đuôi) ───> [30][20][10] ───> poll() (Ra ở đầu hàng đợi)
Phương thức Hành vi xử lý
offer(E e)Thêm phần tử vào cuối hàng đợi
poll()Lấy và xóa phần tử ở đầu hàng đợi. Trả về null nếu trống.
peek()Chỉ xem phần tử ở đầu hàng đợi mà không xóa.
Ví dụ thực tế trong dự án

Ứng dụng làm hàng đợi lưu tạm các email cần gửi của hệ thống để tiến hành gửi tuần tự ngầm tránh nghẽn luồng:

Queue<EmailTask> emailQueue = new LinkedList<>();
emailQueue.offer(new EmailTask("user1@gmail.com"));
emailQueue.offer(new EmailTask("user2@gmail.com"));

// Xử lý gửi:
while(!emailQueue.isEmpty()) {
    EmailTask task = emailQueue.poll();
    sendEmail(task); // Gửi tuần tự
}

11.5 Map (Cấu trúc ánh xạ Key - Value)

Khái niệm và mục đích sử dụng

Map là cấu trúc dữ liệu lưu trữ các cặp thông tin ánh xạ liên kết **Key (Khóa) → Value (Giá trị)**. Trong đó, mỗi Key là duy nhất không trùng lặp, trỏ đến một Value tương ứng.

Các Class triển khai chính:

  • HashMap: Lưu trữ không sắp xếp thứ tự (Tốc độ cao nhất).
  • LinkedHashMap: Giữ nguyên thứ tự nạp thẻ key vào map.
  • TreeMap: Tự động sắp xếp các key theo thứ tự tăng dần.
HashMap hoạt động bên trong như thế nào? (Câu hỏi phỏng vấn Senior cực hot)

HashMap lưu trữ dữ liệu bằng cấu trúc mảng các Buckets. Quy trình lưu trữ cặp (Key, Value):

  1. JVM gọi hàm key.hashCode() để tính toán ra một mã hash nguyên.
  2. Chuyển mã hash này thành chỉ số index của Bucket trong mảng: $\text{Index} = \text{hash} \pmod{\text{Dung lượng mảng}}$.
  3. Nếu Bucket trống: Nạp Node (Key, Value) vào.
  4. Nếu xảy ra **Trùng Bucket (Hash Collision)**:
    • Java sử dụng một danh sách liên kết (Linked List) để nối các Node bị trùng vào cùng bucket.
    • Java 8 Optimization: Nếu danh sách liên kết tại 1 bucket có độ dài vượt quá ngưỡng 8 phần tử, JVM tự động chuyển đổi danh sách này thành một **Cây đỏ đen (Red-Black Tree)** để giảm độ phức tạp tìm kiếm từ $O(N)$ xuống còn $O(\log N)$.
Ví dụ mã nguồn đơn giản
import java.util.HashMap;
import java.util.Map;

public class Main {
    public static void main(String[] args) {
        Map<String, Integer> scores = new HashMap<>();
        scores.put("Toán", 9);
        scores.put("Anh", 8);

        System.out.println(scores.get("Toán")); // In ra: 9
    }
}

11.6 Iterator

Khái niệm và mục đích sử dụng

Iterator là đối tượng cung cấp cơ chế duyệt tuần tự qua các phần tử của mọi Collection một cách an toàn.

Tại sao cần Iterator? (Tránh ConcurrentModificationException)

Nếu bạn thực hiện xóa phần tử trực tiếp của một List khi đang dùng vòng lặp for-each thông thường:

// GÂY RA LỖI CRASH SẬP APP: java.util.ConcurrentModificationException
for (User user : users) {
    if (!user.isActive()) {
        users.remove(user); // Lỗi do cấu trúc danh sách bị thay đổi đột ngột khi đang duyệt
    }
}

Để xóa phần tử an toàn khi đang duyệt danh sách, bạn **bắt buộc** phải sử dụng phương thức remove() của đối tượng Iterator:

Iterator<User> iterator = users.iterator();
while (iterator.hasNext()) {
    User user = iterator.next();
    if (!user.isActive()) {
        iterator.remove(); // Xử lý xóa an toàn tuyệt đối
    }
}

Bản đồ lựa chọn Collection khi đi làm

Nghiệp vụ thực tế Cấu trúc khuyên dùng
Cần lưu danh sách tuần tự, lấy dữ liệu theo vị trí chỉ số indexArrayList
Cần lưu thông tin cấu hình dạng khóa ngoại lệ (Key -> Value), tra cứu nhanhHashMap
Cần lọc trùng lặp các thẻ phân quyền, danh sách email độc nhấtHashSet
Cần làm hàng đợi trung gian gửi tin, xử lý yêu cầu khách hàng vào trước ra trướcQueue (LinkedList / PriorityQueue)

Thống kê tần suất xuất hiện Collection trong dự án Backend thực tế:

  • ArrayList: Chiếm khoảng 70% tần suất code viết.
  • HashMap: Chiếm khoảng 20% tần suất code.
  • HashSet: Chiếm khoảng 5% tần suất.
  • Các loại Collection khác: Chiếm 5% còn lại.
