1.1 Nền tảng CSS cơ bản & Hệ thống biến

Trước khi chuyển sang Tailwind, một nhà phát triển Full-stack cần làm chủ các kỹ thuật cốt lõi của CSS thuần:

Root CSS (:root) và biến CSS

Mã giả lớp :root đại diện cho thẻ cao nhất <html>. Đây là nơi định nghĩa các biến màu sắc, phông chữ toàn cục để tái sử dụng:

:root {
  --accent-cyan: #06b6d4;
  --bg-card: #0d1527;
  --text-primary: #f3f4f6;
}

CSS Reset / Preflight trong Tailwind

Các trình duyệt luôn có khoảng cách lề (margin/padding) mặc định cho các thẻ. Tailwind tự động tích hợp **Preflight** để đưa toàn bộ thiết lập này về không, giúp giao diện đồng bộ trên mọi nền tảng trình duyệt.

Mô hình hộp (Box Model) & Box-Sizing

Thuộc tính box-sizing: border-box cực kỳ quan trọng trong bố cục giao diện hiện đại. Nó đảm bảo kích thước chiều rộng/chiều cao bạn khai báo đã bao gồm cả phần đệm (padding) và đường viền (border), tránh việc giao diện bị đẩy lệch do tăng kích thước ngoài mong muốn.

1.2 Triết lý thiết kế Utility-First

Utility-First là phương pháp viết CSS sử dụng các class đơn chức năng để ghép nối trực tiếp trên thẻ HTML, giúp bạn xây dựng giao diện nhanh chóng mà không cần rời mắt khỏi tệp tin mã nguồn (HTML/React Component).

So sánh BEM truyền thống vs Utility-first:

❌ BEM: Viết class="card-container", sau đó mở file CSS khai báo .card-container { padding: 16px; border-radius: 8px; background: white; }.

✅ Tailwind: Viết trực tiếp: class="p-4 rounded-lg bg-white".

Cơ chế này loại bỏ hoàn toàn việc tốn thời gian suy nghĩ đặt tên class và ngăn ngừa file CSS phình to không kiểm soát khi dự án mở rộng.

1.3 Cấu trúc câu lệnh class Tailwind

Hệ thống class của Tailwind được chuẩn hóa theo công thức tổng quát:

[modifier/prefix]:[property]-[value]

Ví dụ phân loại:

  • Class cơ bản: p-4 (Padding), m-2 (Margin), text-xl (Cỡ chữ), bg-blue-500 (Màu nền).
  • Thiết kế đáp ứng (Responsive): md:text-lg (Từ màn hình cỡ trung sẽ hiển thị cỡ chữ lớn).
  • Trạng thái tương tác (Pseudo): hover:bg-blue-600 (Đổi màu nền khi di chuột qua), disabled:opacity-50 (Mờ đi khi nút bị khóa).
  • Giao diện tối (Dark mode): dark:bg-gray-900, dark:text-white.

2.1 Hệ thống đơn vị đo lường & Spacing

Mặc định, Tailwind sử dụng đơn vị đo lường cơ sở là rem nhằm phục vụ tối ưu cho khả năng tiếp cận (Accessibility - hỗ trợ thu phóng màn hình). Hầu hết các trình duyệt quy định font chữ cơ sở gốc là $1\text{rem} = 16\text{px}$.

Công thức quy đổi chuẩn của Tailwind:

$$1\text{ đơn vị Tailwind} = 0.25\text{rem} = 4\text{px}$$

Ví dụ: Class p-4 tương đương với kích thước khoảng đệm bằng $4 \times 4\text{px} = 16\text{px}$ (tức $1\text{rem}$).

Cú pháp giá trị tùy ý (Arbitrary Values): Khi cần gán kích thước nằm ngoài thiết lập chuẩn, ta sử dụng cú pháp ngoặc vuông: w-[283px] hoặc h-[80vh].

2.2 Padding, Margin, Gap & Space Between

Hệ thống lớp điều chỉnh khoảng cách đệm (Padding) và lề (Margin):

Padding (Khoảng đệm trong)

p-{size} (Tất cả)
px-{size} (Trục X)
py-{size} (Trục Y)
pt-{size} (Trên)
pb-{size} (Dưới)
pl-{size} (Trái)
pr-{size} (Phải)

Ví dụ: <div class="px-6 py-3"></div> tương đương padding trái/phải 24px và trên/dưới 12px.

Margin (Lề khoảng cách ngoài)

m-{size}
mx-{size}
my-{size}
mt-{size} (Lề trên)
mb-{size} (Lề dưới)
mx-auto (Căn giữa)

Gap (Khoảng cách giữa các ô)

Chỉ sử dụng khi bố cục phần tử cha được khai báo hiển thị là flex hoặc grid:

gap-{size} (Tất cả)
gap-x-{size} (Cột)
gap-y-{size} (Hàng)

Space Between (Tạo khoảng cách giữa các phần tử con liên tiếp)

Lớp tiện ích tự động thêm lề ngoài cho tất cả các phần tử con bắt đầu từ phần tử thứ 2:

space-x-{size}
space-y-{size}

4.1 Thiết kế đáp ứng (Responsive Design Breakpoints)

Tailwind áp dụng triết lý thiết kế ưu tiên giao diện di động trước (Mobile-First). Các class không có tiền tố là hiển thị cho thiết bị nhỏ. Khi màn hình lớn dần lên, ta sử dụng các breakpoint sau để tùy biến giao diện:

Tiền tố Kích thước màn hình tối thiểu CSS Media Query tương ứng
sm 640px @media (min-width: 640px) { ... }
md 768px @media (min-width: 768px) { ... }
lg 1024px @media (min-width: 1024px) { ... }
xl 1280px @media (min-width: 1280px) { ... }
2xl 1536px @media (min-width: 1536px) { ... }

Ví dụ: <div class="text-sm md:text-lg lg:text-2xl">

4.2 Giao diện tối (Dark Mode) & Trạng thái tương tác (State Modifiers)

Khai báo Dark Mode trong HTML:

Mặc định, khi kích hoạt chế độ tối qua lớp .dark trên body, các tiền tố dark: sẽ hoạt động:

<div class="bg-white text-gray-900 dark:bg-gray-900 dark:text-white dark:border-gray-700">
  Nội dung tự động phản hồi theo chế độ Light/Dark.
</div>

Trạng thái tương tác (Interactive States):

  • hover:bg-blue-600: Thay đổi màu nền khi người dùng di chuột qua.
  • focus:ring-2 focus:ring-blue-500: Hiển thị viền bao quanh khi nhấp chọn (thường dùng cho các thẻ input trong Form).
  • active:scale-95: Tạo hiệu ứng nút lún nhẹ xuống khi bấm chuột vào.
  • disabled:opacity-50 disabled:cursor-not-allowed: Hiển thị trạng thái vô hiệu hóa của phần tử.

4.3 Tùy biến cấu hình Tailwind (Tailwind Config)

Trong thực tế phát triển dự án, doanh nghiệp thường có bộ mã màu nhận diện thương hiệu riêng. Chúng ta thực hiện mở rộng cấu hình theme mặc định bằng tệp tin tailwind.config.js:

module.exports = {
  theme: {
    extend: {
      colors: {
        brand: {
          50: '#f0f9ff',
          500: '#0ea5e9',
          900: '#0c4a6e',
        }
      }
    }
  }
}

Sau khi khai báo màu sắc thương hiệu trên, Tailwind sẽ tự động tạo ra một hệ thống class tương ứng cho dự án của bạn: bg-brand-500, text-brand-900, border-brand-50...

4.4 JIT Compiler & Tối ưu hóa dung lượng file CSS Production

Từ phiên bản 3.0 trở lên, Tailwind chuyển sang sử dụng trình biên dịch **JIT (Just-In-Time)** mặc định.

Trình biên dịch JIT quét toàn bộ các file mã nguồn của bạn theo thời gian thực khi viết code, chỉ trích xuất và biên dịch đúng những class nào bạn thực tế có viết trong code để tạo ra file CSS đầu ra cực kỳ nhỏ gọn (thường dưới 10KB sau khi nén).

⚠️ Hạn chế viết class động (Dynamic Class Concatenation): JIT quét mã nguồn dưới dạng phân tích chuỗi tĩnh thô, nó sẽ không thể nhận diện được các class sinh ra bằng cơ chế ghép chuỗi động JavaScript.

❌ SAI: class="text-${color}-500"
✅ ĐÚNG: class={color === 'blue' ? 'text-blue-500' : 'text-red-500'}

5.1 Các bộ thẻ component kết hợp thông dụng khi đi làm

Dưới đây là một số tổ hợp lớp tiện ích kinh điển thường được sử dụng khi phát triển ứng dụng web thực tế:

1. Cấu trúc khung bao cục chính (Layout Wrapper)

Thiết lập độ rộng tối đa chuẩn của trang web và căn giữa nội dung:

container mx-auto px-4 max-w-7xl

2. Thẻ hiển thị thông tin (Card Component)

bg-white rounded-xl shadow-md p-6 border border-gray-100

3. Nút bấm tương tác chuẩn (Button Component)

px-4 py-2 bg-blue-500 text-white rounded-lg hover:bg-blue-600 transition duration-300 active:scale-95 disabled:opacity-50

4. Ô nhập liệu trong biểu mẫu (Form Input Field)

w-full border border-gray-300 rounded-lg px-3 py-2 focus:ring-2 focus:ring-blue-500 focus:outline-none transition

5.2 Bộ class cốt lõi cần thuộc lòng

Chỉ cần ghi nhớ và thành thục khoảng 60 class cơ bản dưới đây là bạn đã có thể làm chủ và xây dựng được hơn 90% giao diện dự án thực tế:

Cơ cấu & Bố cục flex | grid | block | hidden
items-center | justify-center | justify-between
gap-2 | gap-4 | gap-6
w-full | h-full | max-w-7xl
relative | absolute | top-0 | right-0 | z-50

m-2 | mt-4 | mb-4 | mx-auto
text-sm | text-base | text-lg | text-xl | text-2xl
font-medium | font-semibold | font-bold | text-center
text-gray-700 | text-white | bg-white | bg-blue-500

5.3 Lộ trình học thực chiến dành cho Fullstack Developer

Để tối ưu hóa thời gian học tập, bạn hãy tập trung nghiên cứu theo lộ trình tích lũy kiến thức sau:

1 Làm chủ Spacing & Sizes: Thuộc các lớp padding, margin, gap và công thức quy đổi pixel/rem.
2 Bố cục trang nâng cao: Thành thục cách dựng layout bằng Flexbox và Grid (grid-cols-12, justify-between).
3 Responsive & Dark Mode: Thiết kế thích ứng trên mọi màn hình sử dụng tiền tố md, lg và xây dựng Dark Mode.
4 Tự chỉnh cấu hình & Kết hợp thư viện UI: Cách viết file tailwind.config.js và tìm hiểu các thư viện UI ăn liền cực mạnh hiện nay như DaisyUI hoặc Shadcn UI để tăng tốc độ làm dự án thực tế.
