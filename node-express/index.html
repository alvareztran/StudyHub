1.1 NodeJS Core & Kiến trúc Event Loop

NodeJS là một môi trường chạy mã JavaScript (Run-time environment) phía máy chủ, được xây dựng trên bộ máy JavaScript V8 hiệu năng cao của Google Chrome.

Kiến trúc Single-threaded & Non-blocking I/O

Khác với các công nghệ backend truyền thống như Java (Tomcat) hay PHP (Apache) vốn sử dụng mô hình đa luồng (Multi-threaded - mỗi Request từ Client sẽ mở ra một luồng xử lý riêng của CPU), NodeJS hoạt động trên cơ chế **Single-threaded** (đơn luồng) kết hợp với **Event Loop** (Vòng lặp sự kiện).

Mô hình xử lý luồng công việc:

  1. Khi có Request gửi tới, NodeJS nhận yêu cầu và xếp vào hàng đợi sự kiện (Event Queue).
  2. Single thread chính sẽ liên tục quét hàng đợi này. Đối với các tác vụ tốn thời gian như đọc file (I/O), truy vấn Database hoặc gọi API ngoài, thread chính sẽ không đứng đợi mà đẩy tác vụ đó xuống cho hệ thống nhân hệ điều hành xử lý (thông qua thư viện Libuv C++).
  3. Khi tác vụ hoàn thành, hệ thống sẽ gửi một sự kiện kèm dữ liệu phản hồi vào Callback Queue để luồng chính tiếp tục trả kết quả về cho Client.

💡 Ưu thế của NodeJS: Xử lý cực kỳ hiệu quả hàng triệu kết nối đồng thời (Concurrent Connections) có mức tiêu tốn tài nguyên I/O cao (như ứng dụng Chat, Real-time telemetry, Streaming). Tuy nhiên, NodeJS không phù hợp cho các bài toán tính toán CPU nặng (như giải mã video, AI training) vì nó sẽ làm nghẽn luồng xử lý đơn duy nhất này.

1.2 Quản lý xử lý bất đồng bộ (Callbacks, Promises, Async/Await)

Vì NodeJS hoạt động theo cơ chế Non-blocking, việc quản lý luồng dữ liệu bất đồng bộ là kiến thức bắt buộc:

1. Callback Pattern & Callback Hell

Là cơ chế sơ khai nhất, đưa hàm làm đối số. Khi lồng nhau quá nhiều sẽ gây khó đọc (Callback Hell):

fs.readFile('file1.txt', (err, data1) => {
  fs.readFile('file2.txt', (err, data2) => {
    // Lồng nhau quá nhiều lớp...
  });
});

2. Promises & Async/Await

Async/Await giúp viết mã bất đồng bộ trông giống như mã đồng bộ (Synchronous) cực kỳ trực quan và sạch sẽ:

const fs = require('fs').promises;

async function readAllFiles() {
  try {
    const data1 = await fs.readFile('file1.txt', 'utf-8');
    const data2 = await fs.readFile('file2.txt', 'utf-8');
    console.log(data1, data2);
  } catch (error) {
    console.error('Lỗi khi đọc file:', error);
  }
}

2.1 NodeJS Built-in Modules (fs, path, http)

NodeJS cung cấp một số mô-đun lõi tích hợp sẵn để tương tác trực tiếp với phần cứng máy chủ:

Mô-đun Path: Quản lý đường dẫn tệp tin

const path = require('path');
// Ghép nối đường dẫn tuyệt đối an toàn trên mọi hệ điều hành (Windows, Linux)
const filePath = path.join(__dirname, 'uploads', 'images', 'avatar.png');

Mô-đun Fs (File System): Thao tác đọc/ghi tệp

const fs = require('fs');

// Đọc file bất đồng bộ (Non-blocking)
fs.readFile('readme.md', 'utf-8', (err, data) => {
  if (err) throw err;
  console.log(data);
});

Mô-đun HTTP: Khởi tạo Server thô

const http = require('http');

const server = http.createServer((req, res) => {
  res.writeHead(200, { 'Content-Type': 'text/plain' });
  res.end('Hello from raw NodeJS server!');
});

server.listen(3000, () => {
  console.log('Server running on port 3000');
});

2.2 Quản lý thư viện bên ngoài với NPM & Package.json

NPM (Node Package Manager) là kho lưu trữ và quản lý mã nguồn mở lớn nhất thế giới của JavaScript.

Tệp package.json

Chứa toàn bộ siêu dữ liệu (metadata), các câu lệnh chạy nhanh (Scripts) và danh sách thư viện phụ thuộc (Dependencies) của dự án.

{
  "name": "my-backend-app",
  "version": "1.0.0",
  "scripts": {
    "start": "node index.js",
    "dev": "nodemon index.js"
  },
  "dependencies": {
    "express": "^4.18.2"
  },
  "devDependencies": {
    "nodemon": "^3.0.1"
  }
}

Sự khác biệt giữa Dependency phân loại:

  • dependencies: Thư viện cần thiết khi chạy ứng dụng thực tế trên môi trường production (ví dụ: express, mongoose).
  • devDependencies: Các công cụ chỉ phục vụ quá trình viết code ở máy lập trình viên (ví dụ: nodemon tự động tải lại code khi file thay đổi, jest để viết unit test).

3.1 ExpressJS - Thiết lập khung ứng dụng Web

Việc viết server bằng mô-đun HTTP thô của Node.js rất tốn công sức khi xử lý định tuyến (Routing). **ExpressJS** là thư viện web framework phổ biến nhất giúp tối giản hóa công việc này.

Khởi tạo server Express cơ bản:

const express = require('express');
const app = express();
const PORT = process.env.PORT || 5000;

// Middleware giúp Express đọc được dữ liệu JSON gửi lên trong body request
app.use(express.json());

app.get('/', (req, res) => {
  res.json({ message: 'Chào mừng bạn đến với ExpressJS API!' });
});

app.listen(PORT, () => {
  console.log(`Server is running on port ${PORT}`);
});

3.2 Định tuyến ExpressJS (Routing, Request & Response)

Express cung cấp các phương thức tương ứng với các phương thức HTTP Request tiêu chuẩn (GET, POST, PUT, DELETE):

// 1. Lấy dữ liệu qua URL Parameters (req.params)
app.get('/users/:id', (req, res) => {
  const userId = req.params.id;
  res.send(`Lấy thông tin user có ID: ${userId}`);
});

// 2. Lấy bộ lọc qua Query String parameters (req.query)
// Ví dụ: /products?category=laptop&sort=desc
app.get('/products', (req, res) => {
  const { category, sort } = req.query;
  res.json({ category, sort });
});

// 3. Tạo mới dữ liệu nhận từ Request Body (req.body)
app.post('/products', (req, res) => {
  const newProduct = req.body;
  res.status(210).json({
    message: 'Tạo sản phẩm thành công',
    data: newProduct
  });
});

4.1 Kiến trúc Middleware

Middleware là trái tim của ExpressJS. Bản chất Middleware là một hàm trung gian nằm giữa Request của Client và Response cuối cùng của Router. Mỗi request đi qua server sẽ lần lượt đi qua các middleware này.

Cấu trúc của một Middleware hàm:

function myMiddleware(req, res, next) { ... }

Trong đó:
- req, res: Các đối tượng Request và Response tiêu chuẩn.
- next: Hàm điều khiển. Khi được gọi, request sẽ được chuyển tiếp sang middleware hoặc router tiếp theo. Nếu không gọi next(), request sẽ bị treo vô tận tại đây.

Các loại Middleware phổ biến:

  1. Built-in: Tích hợp sẵn của Express (như express.json(), express.static()).
  2. Third-party: Các thư viện cài thêm (như cors mở khóa bảo mật truy cập chéo, helmet chống tấn công XSS, morgan ghi lại log truy cập).

4.2 Viết Custom Middleware thực tế (Logger & Auth check)

Ví dụ viết middleware tự động ghi lại lịch sử truy cập (Logger) của client và kiểm tra quyền admin:

// 1. Logger Middleware
const requestLogger = (req, res, next) => {
  const time = new Date().toISOString();
  console.log(`[${time}] ${req.method} - Đường dẫn: ${req.url}`);
  next(); // Cho phép request đi tiếp
};

// Áp dụng logger cho toàn bộ hệ thống
app.use(requestLogger);

// 2. Auth Guard Middleware
const requireAdmin = (req, res, next) => {
  const { role } = req.headers;
  if (role === 'admin') {
    next(); // Hợp lệ, đi tiếp tới API
  } else {
    res.status(403).json({ error: 'Quyền truy cập bị từ chối!' });
  }
};

// Áp dụng bảo vệ riêng cho router nhạy cảm
app.delete('/database/clean', requireAdmin, (req, res) => {
  res.send('Đã dọn dẹp cơ sở dữ liệu.');
});

5.1 Thiết kế RESTful API chuẩn và cấu trúc thư mục MVC

Mô hình MVC (Model - View - Controller) chia cấu trúc dự án backend thành các lớp riêng biệt để dễ dàng bảo trì và mở rộng khi làm việc nhóm:

Cấu trúc thư mục khuyên dùng cho dự án Node/Express:

📂 my-express-app
 ├── 📂 config          # Kết nối Database, cấu hình môi trường (.env)
 ├── 📂 controllers     # Chứa logic nghiệp vụ điều khiển
 ├── 📂 models          # Định nghĩa cấu trúc dữ liệu (Database Schema)
 ├── 📂 routes          # Định nghĩa các tuyến đường dẫn API
 ├── 📂 middlewares     # Middleware tự định nghĩa (Auth, Logger)
 ├── 📄 app.js          # Khởi chạy Express và nạp các middleware
 └── 📄 server.js       # Điểm khởi động ứng dụng (Listen port)

Ví dụ viết Route & Controller sạch tách biệt:

// File: routes/userRoutes.js
const express = require('express');
const router = express.Router();
const userController = require('../controllers/userController');

router.get('/', userController.getAllUsers);
module.exports = router;

// File: controllers/userController.js
exports.getAllUsers = async (req, res) => {
  try {
    const users = await UserModel.find();
    res.status(200).json(users);
  } catch (error) {
    res.status(500).json({ message: 'Lỗi máy chủ' });
  }
};

5.2 Xử lý lỗi tập trung trong ExpressJS

Thay vì viết khối try-catch và tự trả về lỗi `500` ở từng controller, ExpressJS cho phép bạn ném lỗi ra ngoài và xử lý tại một Middleware xử lý lỗi duy nhất nằm ở cuối tệp tin cấu hình.

Để chỉ định đó là middleware xử lý lỗi, hàm callback phải có đủ **4 tham số đầu vào**:

// Middleware xử lý lỗi tập trung (Error Handling Middleware)
app.use((err, req, res, next) => {
  console.error(err.stack); // Ghi lại lỗi log ra server console

  const statusCode = err.statusCode || 500;
  res.status(statusCode).json({
    success: false,
    message: err.message || 'Lỗi hệ thống không xác định!',
    // Chỉ hiển thị stack trace của lỗi ở môi trường dev để bảo mật
    stack: process.env.NODE_ENV === 'development' ? err.stack : {}
  });
});

6.1 Tích hợp cơ sở dữ liệu NoSQL (MongoDB & Mongoose)

MongoDB là cơ sở dữ liệu dạng tài liệu (Document-based) cực kỳ phổ biến kết hợp với Node.js. Thư viện Mongoose đóng vai trò làm ORM/ODM giúp định nghĩa Schema chặt chẽ cho dữ liệu.

Mã nguồn kết nối và định nghĩa Model:

const mongoose = require('mongoose');

// Kết nối MongoDB
mongoose.connect('mongodb://localhost:27017/studyhub_db')
  .then(() => console.log('Đã kết nối MongoDB thành công!'))
  .catch(err => console.error('Lỗi kết nối:', err));

// Định nghĩa lược đồ dữ liệu (Schema)
const productSchema = new mongoose.Schema({
  name: { type: String, required: true },
  price: { type: Number, required: true },
  inStock: { type: Boolean, default: true }
});

const Product = mongoose.model('Product', productSchema);

6.2 Tích hợp cơ sở dữ liệu SQL (PostgreSQL & Sequelize / Prisma)

Đối với cơ sở dữ liệu quan hệ (SQL) như PostgreSQL hay MySQL, Prisma là bộ công cụ ORM thế hệ mới được khuyên dùng nhất hiện nay nhờ khả năng gõ kiểu mạnh (Type-safe).

Sử dụng Prisma Client trong Express Controller:

const { PrismaClient } = require('@prisma/client');
const prisma = new PrismaClient();

exports.getUserPosts = async (req, res) => {
  const userId = parseInt(req.params.userId);
  try {
    const posts = await prisma.post.findMany({
      where: { authorId: userId },
      include: { author: true } // Tự động JOIN liên kết bảng User
    });
    res.json(posts);
  } catch (error) {
    res.status(500).json({ error: error.message });
  }
};

7.1 Bảo mật tài khoản (Bcrypt mật khẩu & Cấp mã JWT)

An ninh hệ thống đòi hỏi mật khẩu của người dùng không bao giờ được lưu dưới dạng văn bản thô (Plain text) trong cơ sở dữ liệu.

1. Mã hóa mật khẩu với Bcrypt

Bcrypt sử dụng giải thuật băm một chiều (One-way hash) kèm Salt (muối) để mã hóa mật khẩu cực kỳ an toàn:

const bcrypt = require('bcrypt');

// Đăng ký: Băm mật khẩu để lưu vào DB
const saltRounds = 10;
const hashedPassword = await bcrypt.hash(rawPassword, saltRounds);

// Đăng nhập: Đối chiếu mật khẩu người dùng nhập vào với DB
const isMatch = await bcrypt.compare(loginPassword, user.password);

2. Cấp phát Token xác thực JSON Web Token (JWT)

Sau khi đăng nhập thành công, server sẽ cấp cho client một chuỗi mã JWT đã được ký số bằng mã bí mật ở phía server:

const jwt = require('jsonwebtoken');

// Tạo token chứa ID người dùng và có hạn sử dụng 1 ngày
const token = jwt.sign(
  { id: user._id, role: user.role },
  process.env.JWT_SECRET,
  { expiresIn: '24h' }
);

7.2 Xây dựng Middleware bảo vệ Route truy cập

Để ngăn chặn người dùng chưa đăng nhập truy cập các tài nguyên nhạy cảm, ta viết một middleware xác thực token JWT gửi kèm lên tiêu đề header:

const jwt = require('jsonwebtoken');

const authGuard = (req, res, next) => {
  // Lấy token từ header Authorization (Thường ở dạng: Bearer )
  const authHeader = req.headers['authorization'];
  const token = authHeader && authHeader.split(' ')[1];

  if (!token) {
    return res.status(401).json({ error: 'Không tìm thấy token. Truy cập bị từ chối!' });
  }

  try {
    const verified = jwt.verify(token, process.env.JWT_SECRET);
    req.user = verified; // Lưu trữ thông tin đã giải mã vào req.user
    next();
  } catch (error) {
    res.status(403).json({ error: 'Mã token không hợp lệ hoặc đã hết hạn!' });
  }
};

// Áp dụng bảo vệ route
app.get('/dashboard', authGuard, (req, res) => {
  res.send(`Xin chào User có ID: ${req.user.id}`);
});

8.1 Môi trường Production & Quản lý tiến trình PM2

Khi đẩy dự án NodeJS lên máy chủ chạy thực tế (Production VPS/Cloud), việc dùng lệnh chạy thô node app.js có hai nhược điểm cực lớn: server sẽ sập ngay lập tức nếu xảy ra lỗi ngoại lệ chưa bắt (Unhandled exception) và không tận dụng được sức mạnh đa nhân của CPU.

Trình quản lý tiến trình PM2

PM2 giúp khắc phục triệt để bằng cơ chế giám sát tự động khởi động lại và cân bằng tải đa nhân CPU (Cluster mode).

Các câu lệnh PM2 thiết yếu khi đi làm:

  • npm install pm2 -g: Cài đặt PM2 toàn cục hệ thống.
  • pm2 start server.js --name "my-app": Khởi chạy dự án.
  • pm2 start server.js -i max: Chạy ở chế độ **Cluster Mode**, tự động nhân bản server tương ứng với số lõi CPU của VPS để tối đa hiệu năng.
  • pm2 list: Liệt kê danh sách các server đang chạy ngầm.
  • pm2 logs: Xem log lỗi và truy cập trực tiếp thời gian thực.
  • pm2 stop/restart/delete all: Điều khiển tắt, bật lại hoặc xóa tiến trình.

8.2 Lộ trình phát triển Fullstack Backend thực tế

Để trở thành một lập trình viên backend NodeJS vững tay nghề, bạn nên học tập theo trình tự lũy tiến sau:

1 Lõi Javascript & Built-in module: Nắm vững Event Loop và xử lý đọc/ghi file, luồng dữ liệu (Streams).
2 Thiết kế RESTful API với ExpressJS: Biết phân chia Routing, viết Controller, làm chủ Middleware và cơ chế bắt lỗi tập trung.
3 Làm việc với Cơ sở dữ liệu: Tạo các thao tác CRUD và thiết lập liên kết dữ liệu quan hệ (Prisma/SQL) hoặc phi quan hệ (Mongoose/MongoDB).
4 An ninh & Bảo mật: Thực thi xác thực phiên làm việc người dùng bằng mã token JWT, băm mật khẩu bcrypt và thiết lập quyền truy cập.
5 Mở rộng nâng cao đi làm: Tìm hiểu về Socket.io (Real-time), lập lịch tác vụ (Cron job), cấu hình caching Redis và làm quen với framework cấp tiến NestJS (TypeScript).
